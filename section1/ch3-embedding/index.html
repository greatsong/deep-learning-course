    3차시: 단어로 산수를 할 수 있다고? (임베딩)¶

    이번에 배울 것¶

    • [x] 컴퓨터가 단어를 숫자로 바꾸는 방법을 알아요
    • [x] 임베딩이 무엇인지 설명할 수 있어요
    • [x] 단어 벡터로 산수가 되는 원리를 이해해요
    • [x] AI가 단어의 의미를 어떻게 학습하는지 알아요

    도입: 단어로 수학을?¶

    한번 맞춰 봐요!

    왕 - 남자 + 여자 = ?

    정답은... 여왕!

    서울 - 한국 + 일본 = ?

    정답은... 도쿄!

    이게 진짜 된다고요? 네, 진짜 돼요! AI 안에서는 단어가 좌표(숫자)로 표현되어 있어서, 덧셈과 뺄셈이 가능해요. 오늘은 이 마법 같은 원리를 파헤쳐 볼 거예요.

    개념 1: 컴퓨터에게 단어는 그냥 번호?¶

    사전 순서의 한계¶

    컴퓨터는 숫자만 이해해요. 그래서 단어에 번호를 붙여야 해요.

    가장 단순한 방법은 사전 순서대로 번호를 매기는 거예요:

    가방 = 1
강아지 = 2
고양이 = 3
나무 = 4
바나나 = 5
사과 = 6
...

    문제점!

    • "강아지(2)"와 "고양이(3)"는 둘 다 동물이라 의미가 비슷한데, 번호만 보면 그걸 알 수 없어요.
    • "강아지(2)"와 "가방(1)"이 번호상 더 가까운데, 의미는 전혀 다르잖아요!
    • 번호의 크기 자체에는 아무 의미가 없어요.

    이 방식으로는 컴퓨터가 단어의 의미를 절대 이해할 수 없어요.

    개념 2: 단어에 좌표를 주자 = 임베딩¶

    교실 좌석 배치를 떠올려 봐요¶

    선생님이 자리 배치를 할 때, 비슷한 친구들끼리 가까이 앉히는 경우가 있잖아요?

    임베딩도 같은 원리예요. 비슷한 의미의 단어를 가까운 좌표에 배치하는 거예요!

    ← 1 ─ 2 ─ 3 ─ 4 ─ 5 ─ 6 →
  가방 강아지 고양이 나무 바나나 사과

    한 줄 위에 놓으니까, 강아지와 바나나의 관계를 표현할 방법이 없어요.

              생물 ↑
               │  ★강아지  ★고양이
               │
               │      ★나무
               │
     ─────────┼──────────────→ 크기
               │
               │  ★가방
               │     ★사과  ★바나나
          무생물│

    2차원으로 하니까 동물끼리 가깝고, 과일끼리 가까워요! 의미가 좌표에 담기기 시작해요.

    **핵심 개념: 임베딩(Embedding)** 단어를 **의미가 담긴 좌표(숫자 묶음)**로 바꾸는 것을 **임베딩**이라고 해요. 비슷한 의미의 단어는 가까운 좌표에, 다른 의미의 단어는 먼 좌표에 놓여요.
    더 알아보기: 실제로는 몇 차원?

    위 예시에서는 2차원(x, y)을 사용했지만, 실제 AI에서는 768차원 이상을 사용해요! 차원이 많을수록 단어의 다양한 의미 관계(동의어, 반의어, 상위-하위 개념 등)를 더 정밀하게 표현할 수 있어요.

    예를 들어 "사과"라는 단어를 768개의 숫자로 표현해요:

    사과 = [0.23, -0.45, 0.78, 0.12, ..., -0.33]  (768개의 숫자)

    768차원을 상상하기는 불가능하지만, 원리는 2차원과 같아요!

    개념 3: 왜 산수가 되는가¶

    화살표(벡터)로 생각해 봐요¶

    좌표를 화살표로 생각하면, 산수가 왜 되는지 이해할 수 있어요.

    "왕"에서 "남자"를 빼면, "남성성"이라는 의미가 빠지고 "왕족/지배자"라는 의미만 남아요. 거기에 "여자"를 더하면? "여성 + 왕족" = "여왕" 근처 좌표가 돼요!

        여왕 ← ← ← ← ← ← ← ← ← ← ← ← ← ←
     ↑                                    ↑
     │ + 여자                        + 여자 │
     │                                    │
    왕족/지배자                        왕족/지배자
     ↑                                    ↑
     │ - 남자                        - 남자 │
     │                                    │
     왕                                   왕

    좀 더 쉽게 정리하면:

    연산 의미
    왕 - 남자 "남성"이라는 의미 제거 → "왕족" 개념만 남음
    (왕 - 남자) + 여자 "왕족" + "여성" → "여왕" 근처!

    같은 원리로:

    연산 결과 빠지는 의미 → 더해지는 의미
    서울 - 한국 + 일본 도쿄 "한국의 수도" → "일본의 수도"
    걷다 - 현재 + 과거 걸었다 현재 시제 → 과거 시제
    삼촌 - 남자 + 여자 이모 남성 친척 → 여성 친척

    벡터 연산의 핵심

    단어 사이의 관계가 방향으로 표현돼요. "남자 → 여자" 방향과 "왕 → 여왕" 방향이 같은 방향이에요. 그래서 산수가 되는 거예요!

    개념 4: 어떻게 이런 좌표를 만드나¶

    AI가 어떻게 이렇게 정교한 좌표를 만들 수 있을까요?

    분포 가설: 함께 쓰이는 단어는 비슷하다¶

    "함께 자주 쓰이는 단어는 비슷한 의미를 가진다"

    이것을 분포 가설(Distributional Hypothesis)이라고 해요.

    예를 들어볼게요:

    • "나는 강아지를 산책시켰어요"
    • "나는 고양이를 안아줬어요"
    • "나는 강아지에게 간식을 줬어요"
    • "나는 고양이에게 밥을 줬어요"

    "강아지"와 "고양이"는 비슷한 문장 속에서 비슷한 위치에 등장해요. AI는 이걸 보고 "아, 이 두 단어는 비슷한 의미겠구나"라고 판단하고 가까운 좌표에 배치해요.

    1. 인터넷에서 수십억 개의 문장을 수집해요
    2. 각 단어가 어떤 단어들과 함께 등장하는지 통계를 내요
    3. 비슷한 맥락에서 등장하는 단어끼리 가까운 좌표에 놓여요
    4. 이 과정을 수백만 번 반복하면, 신기하게도 의미 관계가 좌표에 담겨요

    사람이 직접 가르치지 않아요

    "강아지는 동물이야"라고 AI에게 알려주지 않아도, 수많은 문장을 읽다 보면 AI가 스스로 강아지와 고양이가 비슷하다는 걸 학습해요. 이걸 비지도 학습(Unsupervised Learning)이라고 해요.

    실습 2: 단어 좌표 만들기 (모둠 활동)¶

    활동 순서:

    1. A3 용지에 x축과 y축을 그려요
    2. 축의 의미를 모둠이 직접 정해요 (예: x축 = 크기, y축 = 생물/무생물)

    3. 단어 카드 10개를 좌표 위에 배치해요:

      단어 카드 예시:

자동차, 비행기, 연필, 바다, 태양

    4. 다른 모둠과 비교해요 — 축을 다르게 정하면 배치가 달라져요!

    5. AI의 임베딩 결과와 비교해 봐요 — 어디가 비슷하고 어디가 다른가요?

    문제 정답
    아빠 - 남자 + 여자 = ? ???
    서울 - 한국 + 미국 = ? ???
    과거 - 어제 + 내일 = ? ???

    정답 확인
    문제 정답
    아빠 - 남자 + 여자 = ? 엄마
    서울 - 한국 + 미국 = ? 워싱턴 D.C.
    과거 - 어제 + 내일 = ? 미래
    의사 - 병원 + 법원 = ? 판사 (또는 변호사)

    실제 AI에서는 정확히 이 답이 나오지 않을 수도 있어요. 하지만 비슷한 의미의 단어가 나올 거예요!

    자주 하는 오해¶

    오해 1: AI가 단어의 뜻을 '이해'한다?

    아니에요! AI는 뜻을 이해하는 게 아니라, 단어가 어떤 맥락에서 사용되는지 통계적으로 학습한 거예요. "강아지"의 의미를 아는 게 아니라, "강아지"가 "산책", "간식", "꼬리" 같은 단어들과 자주 함께 나온다는 패턴을 아는 거예요.

    오해 2: 단어 산수는 항상 정확하다?

    아니에요! "왕 - 남자 + 여자 = 여왕"은 유명한 예시지만, 모든 단어 조합에서 이렇게 깔끔하게 되지는 않아요. 특히 추상적인 개념이나 다의어(여러 뜻이 있는 단어)에서는 엉뚱한 결과가 나올 수 있어요.

    오해 3: 차원이 높을수록 무조건 좋다?

    항상 그렇지는 않아요! 차원이 너무 높으면 계산이 느려지고, 불필요한 정보까지 저장될 수 있어요. 적절한 차원 수를 찾는 것도 중요한 연구 주제예요.

    정리¶

    핵심 용어 의미 비유
    임베딩 단어를 의미가 담긴 좌표(숫자 묶음)로 바꾸는 것 교실에서 비슷한 친구끼리 가까이 앉히기
    벡터 방향과 크기를 가진 화살표 (= 좌표) 지도에서의 위치와 이동 방향
    단어 벡터 연산 단어 좌표의 덧셈/뺄셈으로 의미 관계 표현 왕 - 남자 + 여자 = 여왕
    분포 가설 비슷한 맥락에서 등장하는 단어는 비슷한 의미 같은 동아리 친구들은 관심사가 비슷하다
    차원 단어를 표현하는 데 사용하는 숫자의 개수 좌표의 축 수 (2D, 3D, 768D...)
    **오늘의 핵심 한 줄** 단어를 좌표로 바꾸면 의미를 계산할 수 있다.
