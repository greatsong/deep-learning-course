    8차시: AI는 어디를 집중해서 볼까? (Attention)¶

    이번에 배울 것¶

    • [x] 긴 문장을 처리할 때 왜 "집중"이 필요한지 이해해요
    • [x] Attention이 무엇인지 일상 비유로 설명할 수 있어요
    • [x] Query, Key, Value의 역할을 도서관 비유로 이해해요
    • [x] Self-Attention이 문장 속 단어들의 관계를 어떻게 파악하는지 알아요
    • [x] "Attention Is All You Need" 논문이 왜 혁명적이었는지 느껴봐요

    도입: 같은 단어인데 의미가 다르다고?¶

    상황 1: "it"은 누구를 가리킬까?¶

    영어 문장 두 개를 비교해 봐요.

    "The animal didn't cross the street because it was too tired."

    "The animal didn't cross the street because it was too wide."

    두 문장에서 it이 가리키는 대상이 완전히 달라요.

    • 첫 번째 문장: it = animal (동물이 너무 피곤해서)
    • 두 번째 문장: it = street (길이 너무 넓어서)

    단어 하나("tired" vs "wide")가 바뀌었을 뿐인데, "it"의 의미가 확 달라져요. AI가 이걸 구별하려면, 문장 전체를 보면서 어디에 집중해야 할지 판단할 수 있어야 해요.

    상황 2: "은행"이 두 개?¶

    "은행에서 돈을 찾았다."

    "은행나무에서 열매를 찾았다."

    첫 번째 "은행"은 금융기관, 두 번째 "은행"은 나무 열매예요. 글자는 같은데 의미가 완전히 다르죠. 사람은 주변 단어("돈" vs "열매")를 보고 자연스럽게 구별하는데, AI도 이걸 할 수 있을까요?

    오늘 배울 Attention이 바로 이 문제를 해결하는 핵심 기술이에요!

    개념 1: 왜 모든 단어를 동등하게 봐선 안 될까?¶

    긴 문장의 함정¶

    이전 세대 AI(RNN)는 문장을 왼쪽에서 오른쪽으로 한 단어씩 순서대로 읽었어요. 마치 긴 복도를 걸으면서 방을 하나씩 들여다보는 것과 같아요.

    문제는 뭘까요? 앞에서 본 내용을 점점 까먹는다는 거예요.

    시험공부할 때를 생각해 봐요. 교과서를 처음부터 쭉 읽었는데, 마지막 페이지에 도달했을 때 첫 페이지 내용이 기억나나요? 대부분 흐릿해지죠. RNN도 똑같은 문제를 겪었어요.

    "나는 어제 학교에서 점심을 먹고 나서 도서관에 갔는데
 거기서 재미있는 책을 발견해서 한참 읽다가
 집에 돌아와서 숙제를 하고 잠을 잤다."

→ RNN이 "잤다"를 처리할 때, "나는"이 뭐였는지 거의 잊어버려요

    사람은 어떻게 할까?¶

    사람은 긴 글을 읽을 때도 핵심 단어에 자동으로 집중해요.

    "시험에 나올 부분"을 찾을 때, 모든 글자를 똑같이 읽지 않잖아요? 형광펜으로 중요한 부분만 하이라이팅하죠. AI에게도 이런 "형광펜"이 필요했어요.

    개념 2: Attention = 동적 집중¶

    **핵심 개념: Attention** Attention은 "이 단어를 이해하려면, 문장의 **어디를 봐야 할까?**"를 계산하는 메커니즘이에요. 모든 단어를 동등하게 보는 게 아니라, **지금 중요한 단어에 더 많이 집중**해요.

    Attention의 핵심 과정은 이래요:

    1. 각 단어 쌍의 "관련도 점수"를 계산해요
    2. 점수가 높은 단어에 더 많은 가중치를 줘요
    3. 가중치가 높은 단어를 더 많이 참고해서 의미를 파악해요

    아까 "it" 예시로 돌아가 볼게요:

    "The animal didn't cross the street because it was too tired."

"it"의 Attention 점수:
  The      █         3%
  animal   ██████████████████  45%  ← 여기에 집중!
  didn't   ██        5%
  cross    ███       7%
  the      █         3%
  street   █████     12%
  because  ██        5%
  was      ███       8%
  too      ███       7%
  tired    ██        5%

    "it"이 "animal"에 높은 점수를 주고 있어요. AI가 it = animal이라는 걸 파악한 거예요!

    개념 3: Query, Key, Value¶

    Attention의 내부를 조금 더 들여다볼게요. 어려운 수식 대신, 도서관 비유로 이해해 봐요.

    도서관에서 책 찾기¶

    여러분이 도서관에 갔어요. "조선시대 왕"에 대해 알고 싶어요.

    내가 찾고 싶은 것 = "조선시대 왕"

    도서관에 들어가면서 머릿속에 품고 있는 질문이에요.

    각 책의 제목/키워드 = "한국사", "세계사", "수학", "물리", "조선왕조실록"

    책꽂이에 꽂힌 각 책의 라벨이에요. Query와 비교해서 관련 있는 책을 찾아요.

    책의 실제 내용 = 각 책 안에 담긴 정보

    Query와 Key가 매칭되면, 그 책(Value)의 내용을 가져와서 읽어요.

    동작 과정:

    1. "조선시대 왕"(Query)을 들고 책꽂이를 쭉 훑어요
    2. 각 책의 제목(Key)과 비교해요
    3. "한국사" → 관련 있음! "조선왕조실록" → 매우 관련 있음! "수학" → 관련 없음
    4. 관련도가 높은 책의 내용(Value)을 더 많이 참고해요
    Query: "조선시대 왕"

Key별 관련도 점수:
  한국사         ████████████      높음
  조선왕조실록   ██████████████████  매우 높음!
  세계사         ████              중간
  수학           █                 낮음
  물리           █                 낮음

    AI의 Attention도 정확히 이 과정이에요. 문장의 각 단어가 Query가 되어서, 다른 단어들(Key)과의 관련도를 계산하고, 관련 높은 단어의 정보(Value)를 더 많이 가져오는 거예요.

    더 알아보기: 실제 계산은?

    실제 Attention에서는 Query와 Key의 관련도를 내적(dot product)으로 계산해요. 두 벡터가 비슷한 방향을 가리키면 점수가 높아지는 원리예요. 그리고 이 점수들을 softmax라는 함수로 변환해서 합이 1(100%)이 되는 확률로 만들어요. 3차시에서 배운 임베딩 벡터가 여기서 활약하는 거예요!

    개념 4: Self-Attention¶

    **핵심 개념: Self-Attention** Self-Attention은 문장 **안에서** 단어들끼리 서로를 참조하는 거예요. "Self"가 붙은 이유는, 외부가 아니라 **같은 문장 안의 단어들끼리** Attention을 주고받기 때문이에요.

    예시를 볼게요:

    "나는 사과를 먹었다. 그것은 맛있었다."

    "그것"이 무엇을 가리키는지 파악하려면, 같은 문장(또는 이전 문장) 안의 단어들을 봐야 해요.

    "그것"의 Self-Attention:
  나는     ██         5%
  사과를   ██████████████████  50%  ← "그것" = "사과"!
  먹었다   ████████   18%
  그것은   ███        8%
  맛있었다 ████████   19%

    "그것"이 "사과를"에 가장 높은 점수를 줬어요. AI가 문맥을 파악한 거예요!

    Self-Attention 덕분에 문장의 모든 단어가 동시에 다른 모든 단어를 참조할 수 있어요. 앞에서 뒤로 한 단어씩 읽는 게 아니라, 한 번에 전체를 보면서 관계를 파악하는 거예요.

    개념 5: 2017년, "Attention Is All You Need"¶

    2017년, 구글 연구팀이 역사적인 논문을 발표해요.

    • 단어를 순서대로 하나씩 처리 (느림)
    • Attention은 보조 장치로만 사용
    • 긴 문장에서 앞부분을 까먹는 문제 여전히 존재
    • GPU로 병렬 처리가 어려움
    • 순서대로 읽는 과정을 완전히 제거
    • Attention이 주인공으로 승격
    • 모든 단어를 동시에 처리 (빠름!)
    • GPU 병렬 처리에 최적화 → 학습 속도 폭발적 향상

    이 논문의 이름이 바로 "Attention Is All You Need" (Attention이면 충분하다). 이 논문에서 탄생한 구조가 Transformer예요.

    이 논문 하나가 세상을 바꿨어요

    ChatGPT, Gemini, Claude, Llama... 지금 여러분이 쓰는 모든 현대 AI가 이 논문의 Transformer 구조를 기반으로 하고 있어요. 2017년의 이 논문은 AI 역사에서 가장 영향력 있는 논문 중 하나예요.

    실습 안내¶

    실습 1: Attention 시각화 체험¶

    BertViz나 exBERT 같은 도구를 사용해서 AI가 실제로 어디에 주목하는지 시각화해 봐요.

    1. 선생님이 준비한 Attention 시각화 도구에 접속해요
    2. 문장을 입력하면 단어 사이에 연결선이 나타나요
    3. 선이 굵을수록 Attention 점수가 높은 거예요
    4. 아까 배운 "it" 문장을 넣어보고, AI가 정말 올바른 단어에 집중하는지 확인해 봐요

    실습 2: 문장 카드에 화살표 그리기¶

    모둠 활동이에요!

    1. 선생님이 문장 카드 5장을 나눠줘요
    2. 밑줄 친 단어가 문장의 어디에 주목해야 하는지 화살표를 그려요
    3. 모둠원끼리 비교 — "나는 여기를 봐야 한다고 생각해!"
    4. AI의 Attention 히트맵과 비교 — 사람과 AI의 판단이 비슷한가요?

    예시 문장들:

    문장 밑줄 단어
    "철수는 영희에게 꽃을 줬다. 그녀는 기뻐했다." 그녀는
    "고양이가 쥐를 쫓았다. 그것은 빨랐다." 그것은
    "나는 은행에서 돈을 인출했다." 은행
    "은행 열매가 가을에 떨어졌다." 은행
    "그는 배를 타고 바다로 나갔다." 배

    실습 3: "은행" 실험¶

    ChatGPT에게 직접 물어봐요!

    1. "은행에서 돈을 찾았다. 여기서 은행은 무슨 뜻이야?"
    2. "은행나무에서 열매를 찾았다. 여기서 은행은 무슨 뜻이야?"
    3. AI가 문맥에 따라 다르게 답하는지 확인해 봐요
    4. 더 어려운 문장도 만들어서 테스트해 봐요: "은행에서 은행을 먹었다" — AI는 이걸 어떻게 해석할까요?

    자주 하는 오해¶

    오해 1: Attention은 사람의 집중력과 같다?

    비슷한 점이 있지만 같지는 않아요. 사람의 집중력은 감정, 피로, 관심사 등 다양한 요인에 영향을 받지만, AI의 Attention은 순수하게 수학적 계산으로 관련도를 구해요. 비유로는 좋지만, 동일한 메커니즘은 아니에요.

    오해 2: Attention이 있으면 AI가 문장을 완벽히 이해한다?

    아니에요! Attention은 단어 간의 통계적 관련성을 계산하는 거예요. "이해"와는 달라요. AI는 "그것 = 사과"라는 관계를 학습 데이터의 패턴에서 파악한 것이지, 사과가 뭔지 진짜 아는 건 아닐 수 있어요.

    오해 3: Attention은 최신 기술이다?

    Attention의 기본 아이디어는 2014~2015년에 이미 등장했어요. 2017년 "Attention Is All You Need"가 혁명적이었던 건, Attention 만으로 모델을 만들 수 있다는 걸 보여줬기 때문이에요. 아이디어 자체보다 적용 방식이 혁신이었어요.

    정리¶

    핵심 용어 의미 비유
    Attention 단어 간 관련도를 계산해서 중요한 곳에 집중 형광펜으로 중요한 부분 하이라이팅
    Query "내가 알고 싶은 것" 도서관에서 찾고 싶은 주제
    Key 각 단어의 라벨/키워드 책의 제목
    Value 각 단어가 담고 있는 실제 정보 책의 내용
    Self-Attention 같은 문장 안 단어끼리 서로 참조 반 친구들끼리 서로 역할 파악
    Transformer Attention만으로 만든 모델 구조 다음 차시에서 자세히!
    **오늘의 핵심 한 줄** Attention은 문장의 모든 단어를 동시에 보면서, 지금 중요한 단어에 집중하는 메커니즘이에요.
