    도입: 시험 점수가 왜 필요할까?¶

    여러분은 시험을 잘 봤는지 어떻게 알죠? 점수를 보잖아요!

    만약 선생님이 시험지를 돌려주면서 점수 없이 "잘했어" 또는 "못했어"만 말해준다면? 다음에 뭘 고쳐야 할지 알 수 없겠죠.

    AI도 완전히 같은 상황이에요. AI가 학습하려면, "내가 얼마나 틀렸는지"를 정확한 숫자로 알아야 해요. 그래야 뭘 어떻게 고칠지 방향이 잡히거든요.

    오늘은 AI에게 시험 점수를 매기는 공식, 바로 손실함수를 알아볼 거예요.

    개념 1: 왜 "틀린 정도"를 숫자로 만들어야 하나?¶

    "맞다/틀리다"만으로는 부족해요¶

    AI가 고양이 사진을 보고 이렇게 대답했다고 해볼게요:

    AI의 예측 상황
    "개 92% 확신!" 자신감 넘치게 틀렸어요
    "개 51% 확신..." 애매하게 틀렸어요

    둘 다 틀리긴 했지만, 얼마나 틀렸는지가 완전히 달라요.

    • 첫 번째: 완전 자신 있게 틀렸으니, 가중치를 많이 고쳐야 해요
    • 두 번째: 거의 맞출 뻔했으니, 조금만 고치면 돼요
    **핵심 개념: 손실(Loss)이란?** "맞았다/틀렸다"가 아니라, **얼마나 틀렸는지**를 숫자로 표현한 것이에요. 이 숫자가 있어야 AI는 "얼마큼 고쳐야 하는지" 알 수 있어요.

    개념 2: 손실함수(Loss Function)란?¶

    과녁 비유로 이해해 봐요¶

    양궁을 떠올려 보세요.

            🎯
      ╱    ╲
     │  ●   │   ← 정중앙 = 정답
     │     ↗│   ← 화살 = AI의 예측
      ╲    ╱
        ╲╱
    • 정중앙(10점 과녁) = 정답
    • 화살이 꽂힌 위치 = AI의 예측
    • 정중앙에서 화살까지의 거리 = 손실(Loss)

    손실함수는 이 "거리"를 숫자로 바꾸는 공식이에요.

    한 줄 정의

    손실함수(Loss Function) = 정답과 예측 사이의 차이를 숫자로 바꾸는 공식. 이 숫자가 작을수록 AI가 잘 맞춘 거예요.

    화살이 정중앙에 가까울수록 → 손실이 작고 → AI가 잘하고 있다는 뜻이에요.

    개념 3: 대표적인 손실함수 (직관적으로!)¶

    MSE (평균 제곱 오차) — "차이를 제곱해서 벌점을 준다"¶

    가장 직관적인 방법이에요. 정답과 예측의 차이를 제곱해요.

    정답: 100점, AI 예측: 95점

    차이 = 100 - 95 = 5
제곱 = 5² = 25   ← 벌점이 작아요

    정답: 100점, AI 예측: 50점

    차이 = 100 - 50 = 50
제곱 = 50² = 2500   ← 벌점이 엄청 커요!

    왜 제곱할까요? 큰 실수를 더 크게 벌주려고예요!

    차이가 2배면, 벌점은 4배. 차이가 10배면, 벌점은 100배. "조금 틀린 건 봐줄게, 근데 크게 틀리면 절대 안 돼!" 라는 느낌이에요.

    더 알아보기: MSE 수식

    MSE = (정답 - 예측)²의 평균

    여러 개의 데이터가 있으면 각각의 (정답 - 예측)²을 구한 다음 평균을 내요. 그래서 평균 제곱 오차라고 불러요. 주로 숫자를 예측하는 문제(키, 몸무게, 온도 예측 등)에서 사용해요.

    Cross-Entropy — "확신하면서 틀리면 최악"¶

    이건 좀 다른 느낌이에요. AI가 자신 있게 틀렸을 때 더 큰 벌을 줘요.

    AI의 예측 실제 정답 벌점 크기
    "고양이 99% 확신!" 고양이 아주 작아요 (거의 0)
    "고양이 51% 확신..." 고양이 좀 있어요
    "고양이 10% 확신..." 고양이 꽤 커요
    "고양이 1% 확신!" 고양이 벌점 폭발!

    마지막 경우를 보세요. AI가 "고양이는 거의 아닐 거야"라고 자신 있게 주장했는데 정답이 고양이였어요. 이때 Cross-Entropy는 엄청난 벌점을 매겨요.

    **MSE vs Cross-Entropy, 한 줄 비교** - **MSE**: 숫자 예측에 적합. "얼마나 차이나?" 를 벌점으로. - **Cross-Entropy**: 분류 문제에 적합. "자신감 있게 틀리면 최악!" 을 벌점으로.

    더 알아보기: Cross-Entropy 직관

    Cross-Entropy는 -log(예측 확률)로 계산해요. log 함수의 특성상, 예측 확률이 0에 가까우면 벌점이 무한대로 치솟아요. 그래서 "확신하면서 틀리는 것"이 최악인 거예요. 주로 분류 문제(고양이/개, 감정 분석 등)에서 사용해요.

    개념 4: 학습 = 손실을 줄이는 과정¶

    AI 학습의 본질은 정말 단순해요. 손실을 줄이는 것, 그게 전부예요!

    Loss 그래프로 보면 확실해요¶

    손실(Loss)
    │
 10 │ ●
    │  ╲
  8 │   ●
    │    ╲
  6 │     ●
    │      ╲
  4 │       ●
    │        ╲╲
  2 │          ●───●───●
    │
  0 ┼──────────────────────→ 학습 횟수(Epoch)
    0   1   2   3   4   5   6   7
    • 학습 초기 (왼쪽): 가중치가 랜덤이라 예측이 엉터리 → 손실이 커요
    • 학습 중반: 점점 패턴을 배워서 → 손실이 줄어들어요
    • 학습 후반 (오른쪽): 꽤 잘 맞추게 됨 → 손실이 작아지고 안정돼요

    학습이 잘 되고 있는지 어떻게 알까?

    Loss 그래프가 우하향(왼쪽 위에서 오른쪽 아래로) 하면 학습이 잘 되고 있는 거예요! 만약 Loss가 줄어들지 않거나 오히려 올라가면, 뭔가 문제가 있다는 신호예요.

    실제 예시: AI가 문장을 만들 때¶

    단계 AI의 출력 손실
    학습 전 "나는 바나나 의자 구름 먹다" 크다
    학습 중 "나는 오늘 학교 갔다 먹었다" 중간
    학습 후 "나는 오늘 학교에서 점심을 먹었다" 작다

    손실이 줄어들수록, AI의 출력이 자연스러워지는 게 보이죠?

    개념 5: 손실함수의 선택이 결과를 바꾼다¶

    심사위원 비유¶

    노래 오디션을 생각해 보세요. 심사위원마다 보는 기준이 달라요:

    • 음정이 정확하면 높은 점수
    • 감정 표현은 별로 안 봄
    • 결과: 기계적으로 정확한 가수가 우승
    • 감정 전달이 좋으면 높은 점수
    • 음정이 좀 틀려도 괜찮음
    • 결과: 감동을 주는 가수가 우승
    • 음정 50% + 감정 30% + 무대 매너 20%
    • 결과: 균형 잡힌 가수가 우승
    **핵심 통찰: 기준이 다르면 결과가 다르다** 손실함수는 AI의 **심사위원**이에요. 어떤 손실함수를 선택하느냐에 따라, AI가 같은 데이터에서도 **완전히 다른 것**을 배워요. 그래서 문제에 맞는 손실함수를 고르는 게 중요해요!

    손실함수 선택은 사람이 해요

    AI가 스스로 "나는 이 손실함수를 쓸래"라고 정하지 않아요. 사람(개발자)이 문제의 성격을 보고 적절한 손실함수를 골라줘야 해요. 이 선택이 AI의 학습 방향을 결정해요.

    실습 안내¶

    실습 1: TensorFlow Playground에서 Loss 관찰하기¶

    1. TensorFlow Playground에 접속해요
    2. 데이터셋을 하나 골라요 (원형 데이터 추천)
    3. ▶ 재생 버튼을 눌러요
    4. 오른쪽 위의 "Test loss"와 "Training loss" 숫자가 줄어드는 걸 관찰해요

    관찰 포인트

    • 학습 초반에 Loss가 빠르게 줄어드나요?
    • 어느 순간부터 Loss가 거의 안 줄어드나요?
    • Loss 그래프의 모양이 위에서 배운 우하향 곡선과 비슷한가요?

    실습 2: 학습률에 따른 Loss 비교¶

    1. TensorFlow Playground에서 Learning rate를 0.001로 설정하고 학습 실행
    2. 리셋 후, Learning rate를 0.1로 바꿔서 다시 학습
    3. 비교해 봐요:
    학습률 Loss가 줄어드는 속도 최종 Loss
    0.001 ? ?
    0.1 ? ?

    어느 쪽이 더 빠르게 줄었나요? (학습률에 대해서는 다음 차시에 자세히 배워요!)

    실습 3: 나만의 손실함수 설계 (모둠 활동)¶

    노래 오디션 심사위원이 되어 볼 거예요!

    상황: 학교 축제 노래 대회 심사를 맡았어요. 점수 항목은 다음 5가지예요.

    항목 설명
    음정 정확도 음을 얼마나 정확하게 맞추는지
    감정 표현 노래에 감정이 실려 있는지
    외모/무대 매너 무대에서 얼마나 자신감 있는지
    관객 반응 관객이 얼마나 열광하는지
    독창성 자기만의 스타일이 있는지

    활동 순서:

    1. 모둠별로 각 항목에 가중치(비중)를 정해요 (합계 100%)
    2. 선생님이 보여주는 같은 공연 영상에 각 모둠의 기준으로 점수를 매겨요
    3. 모둠별 점수를 비교해요
    이 활동의 핵심 깨달음

    같은 공연인데 모둠마다 점수가 다르죠? 이게 바로 손실함수 선택의 효과예요! AI도 마찬가지로, 어떤 기준(손실함수)으로 평가하느냐에 따라 완전히 다른 것을 배워요.

    자주 하는 오해¶

    오해 1: 손실이 0이면 완벽한 AI인가요?

    아니에요! 학습 데이터에서 손실이 0이면, 오히려 과적합(overfitting)일 수 있어요. 시험 문제를 다 외워버린 것처럼, 새로운 문제는 못 푸는 상태예요. 적당한 손실이 오히려 좋은 거예요.

    오해 2: 손실함수는 하나만 있나요?

    수십 가지가 있어요! MSE, Cross-Entropy 외에도 MAE, Hinge Loss, Focal Loss 등 다양해요. 문제의 성격에 따라 다른 손실함수를 사용해요. 마치 상황에 따라 다른 잣대를 쓰는 것처럼요.

    오해 3: AI가 알아서 좋은 손실함수를 고르나요?

    아니에요! 손실함수는 사람이 정해줘요. 그래서 AI 개발자에게 "어떤 손실함수를 쓸 것인가"는 아주 중요한 설계 결정이에요.

    정리¶

    핵심 용어 의미 비유
    손실(Loss) 정답과 예측의 차이를 숫자로 표현한 것 과녁 정중앙과 화살 사이의 거리
    손실함수 손실을 계산하는 공식 시험 채점 기준표
    MSE (정답 - 예측)²의 평균 큰 실수를 더 크게 벌주는 방식
    Cross-Entropy 자신감 있게 틀리면 큰 벌점 "확신하며 틀리면 최악" 방식
    학습 손실을 줄여나가는 과정 시험 점수를 올리려고 공부하는 것
    **오늘의 핵심 한 줄** 틀린 정도를 숫자로 바꾸는 것이 손실함수. 이 숫자를 줄이는 것이 학습.
