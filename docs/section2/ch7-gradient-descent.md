## 도입: 숫자 맞추기 게임

여러분, 모르는 문제가 나오면 어떻게 해요? **감으로 찍어요?**

그런데 만약 이런 게임을 한다고 해볼게요:

!!! note "숫자 맞추기 게임"
    1. 선생님이 **1~100 사이의 숫자** 하나를 생각해요
    2. 여러분이 숫자를 말하면, 선생님은 **"차이(손실)"만** 알려줘요
    3. 정답은 절대 안 알려줘요!

| 시도 | 여러분의 답 | 선생님 피드백 | 해석 |
|:----:|:----------:|:------------:|------|
| 1 | 50 | "차이: 23" | 꽤 멀구나 |
| 2 | 73 | "차이: 0" | 맞았다! |

그런데 잠깐, 2번째에 바로 맞추려면 **방향**을 알아야 하잖아요. "23만큼 차이난다"는 것만으로는 위인지 아래인지 모르니까요.

만약 선생님이 **"23만큼 큰 쪽으로 틀렸어"** 라고 알려준다면? 그러면 50 + 23 = 73으로 바로 갈 수 있어요!

AI의 학습이 바로 이거예요. **정답은 모르지만, "어느 방향으로 얼마나 고쳐야 하는지"는 알 수 있어요.**

## 개념 1: AI는 정답을 모른다 — 방향만 안다

### 눈 감고 산 내려가기

이런 상황을 상상해 보세요:

> 짙은 안개 속에서 산꼭대기에 서 있어요. 앞이 안 보여서 전체 지형은 전혀 모르겠어요. 그런데 **발밑의 기울기**는 느낄 수 있어요. 왼발이 높고 오른발이 낮으면, 오른쪽이 내리막이라는 걸 알 수 있죠.

```
    ☁️ (안개 — 전체 지형은 안 보여요)

        🧑 ← 지금 여기!
       ╱    ╲
      ╱      ╲        "발밑이 왼쪽으로 기울어져 있으니,
     ╱        ╲        왼쪽으로 가면 내려갈 수 있겠다!"
    ╱    ⭐    ╲
   ╱   최저점   ╲
```

**전체 지도가 없어도**, 발밑 기울기만으로 **내리막 방향**을 알 수 있어요. 그 방향으로 한 발짝씩 내디디면, 결국 **골짜기(최저점)**에 도달할 수 있어요.

<div class="key-concept" markdown>

**핵심 비유: 안개 속 하산**

- **산의 높이** = 손실(Loss)의 크기
- **산꼭대기** = 손실이 큰 상태 (학습 초기)
- **골짜기(최저점)** = 손실이 가장 작은 상태 (학습 목표)
- **발밑 기울기** = 기울기(Gradient)
- **한 발짝 내딛기** = 가중치를 조금 수정하기

</div>

## 개념 2: 경사하강법(Gradient Descent)

이제 정식 이름을 붙여볼게요.

- **경사(Gradient)** = 손실이 어느 방향으로 줄어드는지 알려주는 기울기
- **하강(Descent)** = 그 방향으로 내려가기

합치면: **손실이 줄어드는 방향으로 조금씩 이동하는 것** = 경사하강법!

### 1차원으로 쉽게 이해해요

U자 모양의 그릇을 떠올려 보세요. 공을 굴리면 바닥으로 갈 거예요.

```
손실
 │
 │  ●                         ← 출발! (손실 큼)
 │   ╲
 │    ●                       ← 기울기 따라 이동
 │     ╲
 │      ╲
 │       ●                    ← 계속 이동
 │        ╲
 │         ╲  ╱
 │          ●╱                ← 도착! (손실 최소)
 │
 ┼────────────────────→ 가중치 값
```

1. 현재 위치에서 **기울기를 계산**해요 (왼쪽이 내리막? 오른쪽이 내리막?)
2. 내리막 방향으로 **한 발짝 이동**해요 (가중치를 수정해요)
3. 새 위치에서 다시 기울기 계산 → 이동 → 반복!
4. 기울기가 거의 0이 되면 → **최저점 도착!**

!!! note "한 줄 요약"
    **경사하강법** = 손실을 줄이는 방향으로 가중치를 조금씩 수정하는 것. 이걸 수백~수만 번 반복하면 AI가 점점 똑똑해져요.

## 개념 3: 학습률(Learning Rate) — 보폭의 크기

자, 내리막 방향을 알았으면 **얼마나 크게 한 발짝을 내디딜지** 정해야겠죠? 이게 바로 **학습률(Learning Rate)**이에요.

=== "학습률이 너무 클 때 (보폭이 너무 큼)"

    ```
    손실
     │
     │  ●─────────────────●     ← 한 발에 너무 멀리 감!
     │                  ╱   ╲
     │                ╱       ●─────●   ← 왔다 갔다...
     │              ╱                 ╲
     │            ╱                     ●───→ 발산!
     │       ⭐╱
     │     최저점
     ┼────────────────────────────→ 가중치
    ```

    최저점을 **훌쩍 지나쳐서** 반대편으로 가버려요! 왔다 갔다 하다가 오히려 손실이 커지는 **발산** 현상이 일어나요.

=== "학습률이 너무 작을 때 (보폭이 너무 작음)"

    ```
    손실
     │
     │  ●
     │  ●               ← 한 발짝이 너무 작아서...
     │  ●
     │   ●
     │   ●              ← 수업 끝날 때까지 도착 못해요
     │   ●
     │    ●
     │          ⭐
     │        최저점
     ┼────────────────────→ 가중치
    ```

    한 발짝이 너무 작아서 **학습이 엄청 느려요**. 시간이 한참 걸려도 최저점에 도달하지 못해요.

=== "학습률이 적절할 때 (딱 좋은 보폭)"

    ```
    손실
     │
     │  ●
     │    ╲
     │      ●
     │        ╲
     │          ●
     │           ╲
     │            ●⭐     ← 적당한 속도로 도착!
     │          최저점
     ┼────────────────────→ 가중치
    ```

    빠르면서도 정확하게 최저점에 도달해요!

<div class="key-concept" markdown>

**학습률 정리**

| 학습률 | 보폭 | 결과 |
|:------:|:----:|------|
| **너무 크다** | 너무 큼 | 최저점을 지나쳐서 발산 |
| **너무 작다** | 너무 작음 | 수렴은 하지만 너무 느림 |
| **적절하다** | 딱 좋음 | 빠르고 정확하게 수렴 |

</div>

!!! warning "학습률 설정, 쉽지 않아요"
    "적절한" 학습률은 문제마다 달라요. 보통 0.001 ~ 0.01 사이에서 시작하지만, 실험을 통해 찾아야 해요. AI 개발에서 가장 중요한 **하이퍼파라미터** 중 하나예요.

## 개념 4: 함정 — 로컬 미니멈(지역 최솟값)

### W자 지형: 가짜 골짜기에 빠질 수 있다!

실제 손실 함수의 지형은 깔끔한 U자가 아니에요. 울퉁불퉁한 **W자**(또는 더 복잡한 모양)일 수 있어요.

```
손실
 │
 │  ●
 │   ╲
 │    ╲       ╱╲
 │     ╲     ╱  ╲
 │      ╲   ╱    ╲
 │       ╲ ╱      ╲
 │        ●        ╲         ╱
 │    가짜 골짜기     ╲       ╱
 │   (로컬 미니멈)     ╲     ╱
 │                      ╲   ╱
 │                       ╲ ╱
 │                        ⭐
 │                   진짜 최저점
 │                 (글로벌 미니멈)
 ┼──────────────────────────────→ 가중치
```

경사하강법을 쓰면, **가짜 골짜기(로컬 미니멈)**에 빠질 수 있어요. 거기서는 기울기가 0이니까, AI가 "도착했다!"고 착각해요. 하지만 더 깊은 **진짜 최저점(글로벌 미니멈)**이 있을 수 있어요!

### 해결 방법들

=== "모멘텀(Momentum) — 관성 이용"

    공이 내리막을 굴러갈 때 **속도**가 붙잖아요? 그 관성 덕분에 작은 언덕은 넘어갈 수 있어요.

    AI에게도 "이전에 움직이던 방향의 힘"을 기억하게 해서, 작은 가짜 골짜기는 통과하게 만들어요.

=== "여러 출발점에서 시도"

    다른 위치(초기 가중치)에서 여러 번 출발하면, 가짜 골짜기에 빠지지 않는 경로를 찾을 확률이 높아져요.

=== "학습률 변화"

    처음에는 큰 보폭으로 대략적인 위치를 찾고, 나중에는 작은 보폭으로 정밀하게 찾아요. 마치 처음엔 뛰다가 나중엔 살금살금 걷는 것처럼요.

!!! note "더 알아보기: Adam 옵티마이저"
    실제로 많이 쓰이는 **Adam**이라는 방법은, 모멘텀 + 학습률 자동 조절을 합친 것이에요. "방향도 잘 잡고, 보폭도 알아서 조절하는" 똑똑한 하산 전략이라고 생각하면 돼요. 현재 대부분의 딥러닝 학습에서 Adam을 사용해요.

## 개념 5: 역전파(Backpropagation) — 간략히

### 오답 분석의 원리

시험에서 최종 답이 틀렸다고 해볼게요. 여러분은 어떻게 하나요?

> "어디서부터 틀렸지?" → 풀이 과정을 **거꾸로 추적**하잖아요!

AI도 똑같아요.

```
입력 → [1층] → [2층] → [3층] → 출력 → 손실 계산!
                                          │
  "1층, 너 책임 15%"  ←  "2층, 너 책임 30%"  ←  "3층, 너 책임 55%"
                    ← ← ← 역전파 (거꾸로!) ← ← ←
```

1. **출력**에서 손실을 계산해요
2. **거꾸로** 올라가면서, 각 층의 가중치가 얼마나 손실에 기여했는지 계산해요
3. 기여한 만큼 **책임을 분배**해요
4. 각 가중치를 책임만큼 수정해요

<div class="key-concept" markdown>

**역전파(Backpropagation)란?**

출력에서 입력 방향으로 거꾸로 올라가며, 각 가중치가 손실에 얼마나 기여했는지 계산하고, 그만큼 수정하는 방법이에요. "오답 분석을 거꾸로 해서 실수한 곳을 찾는 것"과 같아요.

</div>

!!! tip "왜 '역'전파인가요?"
    보통 신경망은 입력 → 출력 순서로 계산해요 (순전파, Forward). 역전파는 그 반대 방향, 출력 → 입력으로 돌아가며 기울기를 전달해요. 그래서 **역(Back)**전파라고 불러요.

### 실습 2: TensorFlow Playground에서 학습률 실험

[TensorFlow Playground](https://playground.tensorflow.org)에 접속해서 학습률을 바꿔보세요!

| 학습률 | 설정 후 관찰 포인트 |
|:------:|---------------------|
| **0.001** | Loss가 줄어들긴 하는데... 느리죠? |
| **0.03** | 적당한 속도로 줄어들어요 |
| **0.1** | 꽤 빠르게 줄어들어요 |
| **1** | 어라? 좀 불안정해지지 않나요? |
| **10** | Loss가 줄어들지 않거나 오히려 커져요! |

!!! note "스크린샷 비교 활동"
    각 학습률로 **같은 횟수(예: 500 에포크)**만큼 학습한 후, Loss 값과 결과 화면을 스크린샷으로 찍어서 비교해 봐요. 가장 좋은 결과를 낸 학습률은 몇이었나요?

### 실습 3: 등고선 활동지 (종이 활동)

선생님이 나눠주는 **등고선 지도** 활동지를 사용해요.

1. 등고선 지도 위의 **출발점**에 연필을 놓아요
2. 현재 위치에서 **가장 가파른 내리막 방향**으로 화살표를 그려요
3. 화살표 끝에서 다시 방향을 정해요
4. 최저점에 도달할 때까지 반복!

=== "보폭 크게 (학습률 높음)"

    화살표를 **길게** 그려보세요. 최저점을 지나치나요?

=== "보폭 작게 (학습률 낮음)"

    화살표를 **짧게** 그려보세요. 몇 번 만에 도착하나요?

=== "W자 지형에서"

    가짜 골짜기가 있는 지도에서 해보세요. 로컬 미니멈에 빠지나요?

??? success "이 활동의 핵심 깨달음"
    경사하강법은 결국 "가장 가파른 내리막 방향으로 한 발짝 이동"을 반복하는 것이에요. 보폭(학습률)에 따라 결과가 완전히 달라진다는 걸 직접 느껴보는 활동이에요.

## 자주 하는 오해

??? success "오해 1: AI가 정답을 보고 외우는 거 아니에요?"
    **아니에요!** AI는 정답을 보긴 하지만 외우는 게 아니에요. 정답과 예측의 **차이(손실)**를 계산하고, 그 차이를 줄이는 **방향**으로 가중치를 조금씩 고쳐요. 수학 문제의 답을 외우는 게 아니라, **풀이 방법을 개선**하는 거예요.

??? success "오해 2: 경사하강법은 항상 최적의 답을 찾나요?"
    **아니에요!** 로컬 미니멈에 빠질 수 있고, 학습률이 잘못되면 발산할 수도 있어요. 그래서 모멘텀, Adam 같은 개선된 방법들이 계속 연구되고 있어요.

??? success "오해 3: 학습률은 한 번 정하면 끝인가요?"
    **꼭 그렇지는 않아요!** 학습률을 학습 도중에 점점 줄이는 **학습률 스케줄링**이라는 기법도 있어요. 처음에는 크게 움직여서 대략적인 위치를 찾고, 나중에는 작게 움직여서 정밀하게 찾는 전략이에요.

## 정리

| 핵심 용어 | 의미 | 비유 |
|-----------|------|------|
| **경사하강법** | 손실을 줄이는 방향으로 가중치를 조금씩 수정하는 방법 | 안개 속에서 발밑 기울기만 느끼며 산 내려가기 |
| **기울기(Gradient)** | 손실이 줄어드는 방향과 크기 | 발밑의 경사 |
| **학습률** | 한 번에 얼마나 이동할지 결정하는 값 | 한 발짝의 보폭 |
| **로컬 미니멈** | 진짜 최저점이 아닌 가짜 골짜기 | W자 지형의 얕은 골짜기 |
| **역전파** | 출력→입력 방향으로 각 가중치의 책임을 분배하는 방법 | 시험 오답을 거꾸로 추적하는 것 |

<div class="key-concept" markdown>

**오늘의 핵심 한 줄**

기울기 방향으로 조금씩 이동. 보폭이 너무 크면 발산, 너무 작으면 수렴 못한다.

</div>
