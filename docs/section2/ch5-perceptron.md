## 도입: 선 하나로 나눠보기

A4 용지에 점을 찍어 볼게요.

```
빨간 점: (왼쪽 위), (오른쪽 아래)
파란 점: (오른쪽 위), (왼쪽 아래)
```

자, 이 점들을 **직선 하나**로 빨간색 영역과 파란색 영역으로 나눌 수 있을까요?

...어떻게 그어도 안 돼요! 이게 바로 그 유명한 **XOR 문제**예요.

!!! note "역사 속 한 장면"
    **1957년**, 프랭크 로젠블랫이 **퍼셉트론**을 발명했어요. 뉴욕타임스는 "해군이 생각하는 기계의 원형을 공개했다"라고 보도했죠. 사람들은 곧 진짜 인공지능이 올 거라고 흥분했어요.

    그런데 **1969년**, 마빈 민스키와 시모어 페퍼트가 책 한 권을 냈어요. 제목이 바로 『퍼셉트론』. 그 책에서 증명한 것: **"퍼셉트론은 XOR도 못 푼다."**

    이 한 마디가 AI 연구에 찬물을 끼얹었어요. 연구비가 끊기고, 관심이 사라지고... 이 시기를 **"AI 겨울"**이라고 불러요.

---

## 개념 쉽게 이해하기

### 1. 퍼셉트론 = 선 하나 긋기

<div class="key-concept" markdown>

**퍼셉트론의 원리**

```
입력(특징) × 가중치 + 편향 → 기준보다 크면 A, 작으면 B
```

한 줄 요약: **"숫자를 넣으면, 계산해서, 둘 중 하나로 분류한다"**

</div>

스팸 메일 필터로 비유해 볼게요:

=== "입력"
    - "무료"라는 단어가 있나? (있으면 1, 없으면 0)
    - "링크"가 포함되어 있나? (있으면 1, 없으면 0)

=== "가중치"
    - "무료"의 중요도: **×2** (스팸에서 자주 나오니까 가중치가 높아요)
    - "링크"의 중요도: **×1**

=== "계산"
    - "무료" 포함(1) × 2 + "링크" 포함(1) × 1 = **3**
    - 기준(임계값)이 2.5라면? 3 > 2.5 이므로 → **스팸!**

=== "결과"
    - 계산 결과가 기준보다 크면 → 스팸
    - 계산 결과가 기준보다 작으면 → 정상 메일

이걸 2차원 평면으로 그려보면, **직선 하나**로 스팸 영역과 정상 영역을 나누는 거예요. 이것을 **선형 분류**라고 해요.

!!! tip "일상 비유"
    시험에서 "60점 이상이면 합격, 미만이면 불합격" — 이것도 퍼셉트론과 같은 원리예요! 점수라는 입력을 받아서 기준(60점)과 비교해 둘 중 하나로 분류하는 거죠.

---

### 2. 퍼셉트론의 한계: XOR 문제

아까 도입에서 해봤던 것 기억나요? 대각선 위치에 있는 점들은 직선 하나로 나눌 수 없었어요.

```
         파란 ●          빨간 ●

         빨간 ●          파란 ●
```

어떤 방향으로 직선을 그어도, 빨간 점만 한쪽에 모을 수 없어요. 이게 **XOR(배타적 논리합)** 패턴이에요.

!!! danger "왜 이게 큰 문제였을까?"
    XOR은 아주 단순한 문제예요. 입력이 겨우 2개, 점도 겨우 4개인데 못 푸는 거예요. "이렇게 간단한 것도 못 풀면, 복잡한 현실 문제는 어떻게 풀겠어?" — 이게 민스키의 논리였고, 많은 사람이 납득했어요.

---

### 3. 해결책: 층을 쌓자! (다층 퍼셉트론 = 신경망)

그런데 포기하지 않은 사람들이 있었어요. 그들이 발견한 해결책:

<div class="key-concept" markdown>

**선 하나로 안 되면? 여러 개를 조합하면 된다!**

직선 여러 개를 조합하면 곡선처럼 복잡한 경계를 만들 수 있어요.

</div>

```
입력층          은닉층(숨겨진 층)          출력층
┌───┐          ┌───┐
│ × ├────┬────►│ ● ├────┐
└───┘    │     └───┘    │     ┌───┐
         │              ├────►│ ? │ → A 또는 B
┌───┐    │     ┌───┐    │     └───┘
│ × ├────┴────►│ ● ├────┘
└───┘          └───┘
입력            중간 판단         최종 결과
```

- **입력층**: 데이터가 들어오는 곳
- **은닉층(Hidden Layer)**: 중간에서 특징을 조합하는 곳 (눈에 안 보여서 "숨겨진" 층이에요)
- **출력층**: 최종 답을 내는 곳

은닉층의 뉴런 하나하나가 각각 선 하나를 담당해요. 그 선들을 출력층에서 합치면 XOR도 풀 수 있어요!

=== "선 1개 (퍼셉트론)"
    직선 하나 → 단순한 문제만 가능

=== "선 여러 개 (다층 퍼셉트론)"
    직선 여러 개 조합 → 곡선처럼 복잡한 경계 가능

=== "층을 더 깊게 (딥러닝)"
    층이 많아지면 → 아주 복잡한 패턴도 학습 가능 → 이것이 **딥러닝**!

!!! note "'뉴런'이라고 부르는 이유"
    퍼셉트론의 구조가 뇌의 신경세포(뉴런)와 비슷해서 이런 이름이 붙었어요. 신경세포도 여러 입력을 받아서 합산하고, 일정 기준을 넘으면 신호를 보내거든요.

    **하지만!** 실제 뇌의 뉴런은 인공 뉴런보다 훨씬 복잡해요. "영감을 받았다" 정도이지, "뇌를 그대로 모방했다"는 아니에요.

---

### 4. 활성화 함수란?

퍼셉트론에서 "기준보다 크면 켜고, 작으면 꺼"라는 규칙이 있었죠? 이 규칙을 **활성화 함수**라고 해요.

<div class="key-concept" markdown>

**활성화 함수 = 스위치**

계산 결과를 받아서 "이 뉴런을 켤까 말까?" 결정하는 함수예요.

</div>

=== "계단 함수 (초기 버전)"
    ```
    0보다 크면 → 1 (켬)
    0보다 작으면 → 0 (끔)
    ```
    딱 두 가지만 있어서 너무 단순해요.

=== "ReLU (요즘 가장 인기)"
    ```
    음수면 → 0 (끔)
    양수면 → 그 값 그대로 (밝기 조절 가능)
    ```
    켜고 끄는 것뿐 아니라 "얼마나 켤지"도 조절할 수 있어서 훨씬 유용해요!

!!! tip "일상 비유"
    **계단 함수**는 전등 스위치 — ON 아니면 OFF.
    **ReLU**는 조광기(디머 스위치) — OFF이거나, 원하는 만큼 밝기 조절 가능!

## 실습: TensorFlow Playground에서 신경망 설계하기

### 준비물
- 웹 브라우저 (인터넷 연결 필요)

### Step 1: TensorFlow Playground 접속

**[playground.tensorflow.org](https://playground.tensorflow.org)** 에 접속해요.

화면 구성을 먼저 살펴볼게요:

| 영역 | 설명 |
|------|------|
| 왼쪽 위 | 재생 버튼 (학습 시작/정지) |
| 왼쪽 | 데이터셋 선택 (점의 모양) |
| 가운데 | 신경망 구조 (레이어와 뉴런) |
| 오른쪽 | 결과 시각화 (색깔로 분류 영역 표시) |
| 위쪽 | Epoch(학습 횟수), Learning rate(학습률) 등 설정 |

### Step 2: 직선으로 안 되는 걸 확인하기

1. 데이터셋에서 **"Circle (원형)"** 선택
2. Hidden layers를 **0개**로 설정 (HIDDEN LAYERS 옆의 + - 버튼으로 조절)
3. 재생 버튼(▶) 클릭
4. 오른쪽 결과를 관찰 → **직선 하나로는 원 안과 밖을 나눌 수 없어요!**

!!! warning "Loss(손실) 값을 확인하세요"
    오른쪽 위에 보이는 "Test loss" 숫자가 높으면 분류를 잘 못하고 있다는 뜻이에요. 이 숫자가 낮아질수록 잘 분류하고 있는 거예요.

### Step 3: 은닉층 추가하기

1. 리셋 버튼을 눌러 초기화
2. Hidden layers를 **1개**로 설정
3. 뉴런 수를 **2개**로 설정
4. 재생 버튼 클릭 → 결과 관찰
5. 어때요? 원형을 분류할 수 있게 되었나요?

### Step 4: 뉴런 수 늘려보기

1. 리셋 후 뉴런 수를 **4개**로 변경 → 재생 → 관찰
2. 리셋 후 뉴런 수를 **8개**로 변경 → 재생 → 관찰

??? success "관찰 포인트"
    각 뉴런 위에 마우스를 올려보세요! 각 뉴런이 담당하는 영역이 시각화로 보여요. 뉴런마다 다른 "선"을 담당하고 있고, 이것들이 합쳐져서 최종 결과가 나오는 거예요.

### Step 5: 모둠 대결 — "가장 적은 뉴런으로 Spiral 풀기"

이제 진짜 대결이에요!

!!! danger "미션"
    **데이터셋: Spiral (나선형)** — 가장 어려운 패턴!

    **목표: Test Loss가 0.1 이하**

    **승리 조건: 가장 적은 총 뉴런 수로 성공한 모둠이 우승!**

**대결 규칙:**

1. 데이터셋을 **"Spiral"**로 변경
2. 자유롭게 조절할 수 있는 것:
    - Hidden layers 수 (1~6개)
    - 각 레이어의 뉴런 수
    - Learning rate (학습률)
    - Activation function (활성화 함수)
3. 재생 버튼을 눌러 학습 시작
4. Test Loss가 **0.1 이하**가 되면 성공!
5. 성공했을 때의 **총 뉴런 수**를 기록 (모든 은닉층의 뉴런 수 합계)

=== "힌트 1"
    레이어를 2개 이상 쌓아야 할 거예요. Spiral은 정말 복잡한 패턴이거든요.

=== "힌트 2"
    Learning rate가 너무 높으면 학습이 불안정해요. 0.03이나 0.01로 시도해 보세요.

=== "힌트 3"
    활성화 함수를 ReLU로 바꿔 보세요. 기본 Tanh보다 잘 되는 경우가 많아요.

### Step 6: 관찰하고 토론하기

실험이 끝나면 모둠별로 이야기해 봐요:

1. **뉴런이 많으면 무조건 좋을까?** 뉴런이 너무 많으면 어떤 문제가 생길까?
2. **레이어를 깊게 vs 넓게**, 어떤 게 더 효과적이었나?
3. 각 뉴런의 시각화를 보면, **뉴런마다 맡은 역할이 다른 것** 같지 않나요?

## 자주 하는 오해

!!! warning "오해 1: 뉴런이 많을수록 무조건 좋다?"
    뉴런이 너무 많으면 **과적합(Overfitting)**이 일어나요. 학습 데이터는 완벽하게 외워버리지만, 새로운 데이터에는 오히려 못 맞추는 거예요. 시험 문제를 달달 외웠는데, 조금만 바뀌면 틀리는 것과 같아요.

!!! warning "오해 2: 인공 뉴런이 뇌 뉴런과 같다?"
    이름만 같을 뿐, 실제 뇌의 뉴런은 훨씬 복잡해요. 인공 뉴런은 단순한 수학 계산이고, 뇌 뉴런은 화학적·전기적 신호로 작동해요. "영감을 받았다" 정도로 이해하면 돼요.

!!! warning "오해 3: 층이 깊으면 항상 좋다?"
    층이 너무 깊으면 학습이 오히려 어려워져요. 신호가 전달되면서 점점 약해지거든요 (이걸 "기울기 소실"이라고 해요). 그래서 적절한 깊이를 찾는 것도 중요한 기술이에요.

## 정리

<div class="key-concept" markdown>

**핵심 정리**

1. **퍼셉트론** = 선 하나 긋기. 간단한 분류는 가능하지만 XOR 같은 문제는 못 풀어요
2. **다층 퍼셉트론(신경망)** = 선 여러 개 조합. 층을 쌓으면 복잡한 패턴도 분류 가능!
3. **활성화 함수** = 뉴런의 스위치. ReLU가 요즘 가장 인기 있어요
4. **뉴런 하나 = 선 하나. 여러 개 쌓으면 복잡한 문제도 풀린다!**
5. 하지만 **많다고 무조건 좋은 건 아니에요** — 적절한 구조를 찾는 것이 중요해요

</div>
