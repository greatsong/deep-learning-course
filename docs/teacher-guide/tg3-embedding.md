# 3차시 교사용 지도서: 임베딩

## 수업 목표

- 컴퓨터가 단어를 **숫자(벡터)**로 표현한다는 것을 이해한다.
- 의미가 비슷한 단어는 벡터 공간에서 **가까이** 위치한다는 것을 체험한다.
- "왕 - 남자 + 여자 = 여왕" 같은 **단어 산수**가 가능한 이유를 직관적으로 이해한다.

---

## 도입 시나리오 (0~5분)

### 교사 대사

> "칠판에 수학 문제 하나 쓸게요."

(칠판에 크게 쓴다)

```
왕 - 남자 + 여자 = ?
```

> "답이 뭘까요? 3초 줄게요, 머릿속으로..."

(3초 카운트)

> "누가 말해볼래요?"

(학생: "여왕!")

> "맞아요!"

(잠시 멈춤, 진지한 표정으로)

> "그런데... 잠깐. 단어로 뺄셈과 덧셈을 어떻게 하죠? 단어는 숫자가 아닌데?"

(2초 멈춤)

> "비밀은, 컴퓨터에게는 단어가 숫자예요. 더 정확히는 '좌표'예요. 좌표니까 산수가 됩니다."

## 학생 예상 반응 & 교사 피드백

| 학생 반응 | 교사 피드백 |
|-----------|-------------|
| "진짜 여왕이 나와요?" | "네! 실제로 보여드릴게요." (준비된 시연 자료 또는 사전 녹화 영상 재생) |
| "왜 여왕이 나오는 거예요?" | "'왕'의 좌표에서 '남자' 방향을 빼고 '여자' 방향을 더하면, '여왕' 좌표 근처에 도착해요." |
| "좌표가 뭐예요?" | "지도에서 서울의 좌표(위도, 경도) 아시죠? 단어도 비슷하게 숫자 좌표를 가져요." |
| "768차원이 뭐예요?" | "2차원은 x, y 두 개. 3차원은 x, y, z 세 개. 768차원은 특징 768개로 단어를 표현하는 거예요. 상상은 못 하지만 컴퓨터는 계산할 수 있어요." |
| "AI가 단어의 뜻을 아는 건가요?" | "뜻을 '아는' 건 아니에요. 어떤 단어들과 함께 **자주 쓰이는지** 패턴을 학습한 거예요." |
| "다른 것도 돼요? 서울 - 한국 + 일본 = ?" | "좋은 질문! 도쿄가 나와요. 실습 시간에 직접 해볼 거예요." |

## 이론 설명 (5~15분)

### 1단계: 왜 숫자로 바꿔야 할까? (2분)

> "컴퓨터는 글자를 이해 못 해요. 숫자만 알아요. 그래서 단어를 숫자로 바꿔야 하는데, 어떻게 바꾸느냐가 문제예요."

**나쁜 방법 예시:**

```
사과 = 1
바나나 = 2
강아지 = 3
```

> "이렇게 번호를 매기면? '바나나(2) - 사과(1) = 사과(1)'? 말이 안 되죠. 번호에는 의미가 없으니까."

### 2단계: 임베딩이란? (4분)

> "좋은 방법은, 단어를 **의미를 담은 좌표**로 바꾸는 거예요. 이걸 **임베딩(embedding)**이라고 해요. 'embed'가 '끼워 넣다'라는 뜻이에요. 단어를 숫자 공간에 끼워 넣는 거죠."

**칠판/슬라이드 예시 (2차원 단순화):**

```
        맛있다 ↑
             |
    사과 ●   |   ● 바나나
             |
    ─────────┼──────────→ 살아있다
             |
    자동차 ● |   ● 비행기
             |
```

> "보세요. 사과와 바나나는 가까이 있죠? 둘 다 과일이니까. 자동차와 비행기도 가까이 있어요. 둘 다 탈 것이니까. **비슷한 의미의 단어는 가까운 좌표에 위치**해요."

### 3단계: 단어 산수의 원리 (4분)

> "이제 '왕 - 남자 + 여자 = 여왕'이 이해되나요?"

**슬라이드에 화살표 그림:**

```
        왕 ●──── (남자 방향을 빼고) ────→ ● 중간 지점
                                              │
                                    (여자 방향을 더하고)
                                              │
                                              ▼
                                          여왕 ● 근처에 도착!
```

> "'남자'와 '여자'의 차이 벡터, 즉 '성별 방향'이 있어요. '왕'에서 '남자 방향'을 빼고 '여자 방향'을 더하면 '여왕' 근처에 도착하는 거예요."

!!! note "교사 참고: 벡터 설명 깊이"
    "벡터 = 방향과 크기를 가진 화살표"라는 정도만 전달합니다.
    수학적 벡터 연산은 상위 학생에게만 선택적으로 설명하세요.

## 실습 1: 단어 카드 배치 활동 (15~25분)

### 준비물

- **A3 용지** 모둠별 1장 (좌표축을 미리 그려두면 좋음)
- **단어 카드** 모둠별 10~15개

### 단어 카드 목록

!!! info "추천 단어 카드"
    사과, 바나나, 포도, 강아지, 고양이, 토끼, 자동차, 비행기, 자전거, 행복, 슬픔, 분노, 학교, 선생님, 학생

    카드는 포스트잇에 써도 되고, 미리 인쇄해도 됩니다.

### 활동 진행

#### 1단계: 자유 배치 (5분)

> "A3 용지 위에 이 단어 카드들을 배치하세요. 규칙은 하나! **비슷한 단어는 가까이, 다른 단어는 멀리** 놓으세요."

#### 2단계: 축 이름 붙이기 (3분)

> "가로축과 세로축에 이름을 붙여보세요. 어떤 기준으로 나눴나요?"

**학생들이 붙일 수 있는 축 이름 예시:**

- 살아있다 / 살아있지 않다
- 크다 / 작다
- 자연 / 인공
- 감정 / 사물

#### 3단계: 모둠 발표 (5분)

> "왜 그렇게 배치했는지 설명해주세요."

**교사 피드백 포인트:**

| 상황 | 피드백 |
|------|--------|
| 깔끔하게 그룹으로 나눈 경우 | "좋아요! 그런데 '학생'은 '학교'에도 가깝고 '행복'에도 가깝지 않나요? 2차원으로는 이걸 다 표현하기 어렵죠." |
| 축 이름을 못 정한 경우 | "괜찮아요. 실제 임베딩도 축에 딱 떨어지는 이름이 없어요. 그래서 768개나 필요한 거예요." |
| 의외의 배치를 한 경우 | "흥미롭네요! 왜 그렇게 생각했는지 설명해줄래요?" (정답이 없는 활동임을 강조) |

#### 4단계: 핵심 연결 (2분)

> "여러분이 방금 한 게 바로 **임베딩**이에요! 단어를 2차원 좌표에 배치한 거잖아요. AI도 똑같은 일을 해요. 다만 2차원이 아니라 768차원에서요."

## 실습 2: 3D 은하계 탐색 (25~40분)

### micropgt 3D 임베딩 시각화 사용

> "이제 컴퓨터가 만든 임베딩을 직접 봅시다. micropgt의 3D 은하계에서 단어들이 어떻게 배치되어 있는지 탐험해볼 거예요."

#### 탐색 미션

| 미션 | 관찰 포인트 |
|------|-------------|
| **미션 1:** "사과" 검색 → 근처에 어떤 단어가 있나? | 과일류가 모여 있는지 확인 |
| **미션 2:** "행복" 검색 → 근처 단어 관찰 | 감정 단어끼리 모이는지 확인 |
| **미션 3:** "한국" 검색 → 근처 단어 관찰 | 나라 이름끼리 모이는지 확인 |
| **미션 4:** 자유 탐색 — 자신의 관심사 검색 | 예상과 다른 결과가 있는지 기록 |

#### 단어 산수 체험

학생들이 직접 시도해볼 단어 산수:

| 식 | 예상 답 |
|----|---------|
| 왕 - 남자 + 여자 | 여왕 |
| 서울 - 한국 + 일본 | 도쿄 |
| 아침 - 해 + 달 | 밤/저녁 |
| 형 - 남자 + 여자 | 언니/누나 |

> "예상과 다른 결과가 나올 수도 있어요. 그것도 기록해두세요. 왜 다른지 생각해보는 게 중요해요."

### 중위 학생

- 단어 카드 배치 활동 완수 확인

- 활동지에 관찰 결과 기록하는지 확인
- "비슷한 단어끼리 모여 있었나요?"라고 확인 질문

### 하위 학생

!!! warning "지원 방법"
    - **단어 카드 배치 활동을 충분히** 시키세요. 이 활동만으로도 핵심 개념 전달 가능
    - 3D 은하계는 교사가 화면을 보여주며 함께 탐색

    - **직관적 이해가 목표**: "비슷한 단어는 가까이 있다"

## 자주 나오는 질문

!!! question "임베딩은 누가 만들어요?"
    "AI가 학습하면서 자동으로 만들어요. 엄청 많은 텍스트를 읽으면서 '이 단어들은 비슷한 문맥에서 나타나니까 가까이 두자'라고 학습하는 거예요."

!!! question "768차원을 사람이 볼 수 있어요?"
    "직접은 못 봐요. 그래서 특수한 기법(t-SNE, PCA)으로 3차원이나 2차원으로 줄여서 보는 거예요. 3D 은하계가 바로 그거예요. 정보가 약간 손실되지만 대략적인 관계는 볼 수 있어요."

!!! question "모든 단어가 768차원인가요?"
    "모델마다 달라요. GPT-3는 12,288차원, BERT는 768차원이에요. 차원이 높을수록 더 세밀하게 표현할 수 있지만 계산량도 늘어나요."

!!! question "새로운 단어가 나오면 어떻게 해요?"
    "좋은 질문이에요! 새로운 단어는 토큰화(1차시에서 배운 것)를 통해 아는 조각들로 나눠요. 완전히 새로운 단어도 부분적으로는 처리할 수 있어요."

!!! question "단어만 임베딩하나요? 문장은?"
    "문장도 임베딩돼요! 문장 임베딩은 단어 임베딩보다 더 복잡하지만, 원리는 비슷해요. 검색 엔진이 이걸 활용하죠."

!!! question "왕 - 남자 + 여자가 항상 여왕이 나오나요?"
    "항상은 아니에요. 모델과 학습 데이터에 따라 다를 수 있어요. 하지만 대부분의 잘 학습된 모델에서는 여왕이나 비슷한 단어가 나와요."

## 정리 (40~50분)

### 핵심 정리 (5분)

> **오늘 배운 것:**
>
> 1. 컴퓨터는 단어를 **숫자 좌표(벡터)**로 바꿔서 처리한다 → 이것이 **임베딩**
> 2. 의미가 비슷한 단어는 **가까운 좌표**에 위치한다
> 3. 좌표니까 **단어 산수**(왕 - 남자 + 여자 = 여왕)가 가능하다

### 3차시까지의 연결 정리

> "여기서 잠깐, 지금까지 배운 걸 연결해볼게요."

```
입력: "나는 오늘 학교에"
  ↓
1차시: 토큰화 → ["나는", "오늘", "학교", "에"]
  ↓
3차시: 임베딩 → [0.23, -0.15, ...], [0.87, 0.42, ...], ...
  ↓
2차시: 확률 계산 → 갔다(45%), 왔다(25%), ...
  ↓
Temperature 적용 → "갔다" 선택
  ↓
출력: "나는 오늘 학교에 갔다"
```

> "토큰으로 쪼개고 → 숫자로 바꾸고 → 확률로 다음 단어를 고르는 거예요. 이게 ChatGPT가 하는 일의 기본 뼈대예요!"

### 다음 차시 예고

> "그런데 이 과정 중간에 가장 중요한 게 빠져 있어요. 토큰을 숫자로 바꾼 다음, **어떤 계산**을 해서 다음 단어의 확률을 만들어낼까요? 그 계산을 하는 게 바로 **신경망**이에요. 다음 시간에 만나봅시다."
