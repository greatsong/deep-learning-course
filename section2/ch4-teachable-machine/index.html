    4차시: AI야, 이게 뭔지 맞혀봐!¶

    이미지 인식과 Teachable Machine¶

    이번에 배울 것¶

    • [x] AI가 사진을 "숫자"로 본다는 것을 이해해요
    • [x] 분류(Classification)가 뭔지 알아요
    • [x] 학습 데이터가 결과에 미치는 영향을 체험해요
    • [x] Teachable Machine으로 나만의 이미지 분류기를 만들어요
    • [x] CNN(합성곱 신경망)의 기본 아이디어를 맛봐요

    개념 쉽게 이해하기¶

    1. AI는 어떻게 사진을 "보는" 걸까?¶

    우리가 고양이 사진을 보면 "아, 고양이다!" 하고 바로 알잖아요. 그런데 컴퓨터는 사진을 볼 때 우리처럼 의미를 파악하는 게 아니에요. 컴퓨터가 보는 건 숫자 격자예요.

    **사진 = 숫자 격자 (픽셀)** 사진은 아주 작은 점(픽셀)들이 빼곡히 모여 있는 거예요. 각 픽셀에는 **0~255** 사이의 숫자가 들어 있어요. - **0** = 완전히 까만색 - **255** = 완전히 하얀색 - **128** = 중간 회색 컬러 사진이라면? 빨강(R), 초록(G), 파랑(B) 세 장의 숫자 격자가 겹쳐져 있는 거예요!

    예를 들어 100×100 픽셀짜리 사진이 있다면, 컴퓨터가 보는 건 10,000개의 숫자 묶음이에요. 사람은 "고양이의 귀 모양"을 보지만, 컴퓨터는 "이 위치의 숫자들이 이런 패턴으로 배열되어 있구나"를 보는 거예요.

    일상 비유

    모자이크 타일 그림을 떠올려 봐요. 멀리서 보면 그림이 보이지만, 가까이 가면 색깔 타일 하나하나가 보이잖아요? 컴퓨터는 항상 그 타일 하나하나의 색 번호를 보고 있는 거예요.

    2. 분류(Classification)란?¶

    **분류 = 입력을 받아서 카테고리를 정해주는 것** 
    입력(사진) → AI 모델 → 출력(카테고리)
    예시: - 사진 넣기 → AI → "고양이" (95%), "강아지" (5%) - 손 사진 → AI → "주먹" (90%), "보" (10%)

    Teachable Machine이 하는 일이 바로 이거예요. 우리가 보여준 사진들에서 패턴을 찾아내고, 새로운 사진이 들어오면 "이 패턴이랑 비슷하니까 이 카테고리야!"라고 판단하는 거예요.

    마치 친구 얼굴을 외우는 것과 비슷해요. 여러 번 보면서 특징을 기억해두고, 나중에 멀리서 봐도 "아, 저 사람이구나!" 하고 알아보는 거죠.

    3. 데이터가 결과를 바꾼다¶

    이 부분이 정말 중요해요.

    주의! AI는 학습한 것만 안다

    고양이 사진만 잔뜩 학습시키면, AI는 강아지를 봐도 "고양이!"라고 대답해요. AI가 멍청한 게 아니라, 고양이밖에 모르니까 그런 거예요.

    좋은 AI를 만들려면 학습 데이터가 세 가지 조건을 갖춰야 해요:

    조건 설명 예시
    양 충분히 많아야 해요 사진 3장으로는 부족, 30장이면 훨씬 나아요
    다양성 여러 상황을 담아야 해요 밝은 곳, 어두운 곳, 다양한 각도
    품질 정확하게 분류되어 있어야 해요 고양이 폴더에 강아지가 섞이면 안 돼요

    편향(Bias) 문제

    만약 "사람 얼굴" 분류기를 만드는데 한쪽 피부색 사진만 학습시키면? 다른 피부색의 얼굴은 잘 인식하지 못해요. 이게 바로 AI 편향이에요. 실제로 이런 문제 때문에 사회적 논란이 일어난 사례가 많아요.

    4. 필터(Filter)와 특징 추출 — CNN 맛보기¶

    자, 그러면 AI는 숫자 덩어리에서 패턴을 어떻게 찾을까요? 여기서 등장하는 게 필터예요.

    **필터 = 작은 창문으로 이미지를 훑는 것** 손가락으로 프레임을 만들어서 사진 위를 천천히 움직여 본다고 상상해 봐요. 그 작은 창문이 지나가면서 "여기에 선이 있네!", "여기에 모서리가 있네!" 하고 특징을 발견하는 거예요.

    이런 필터를 여러 층으로 쌓으면 점점 복잡한 것을 인식할 수 있어요:

    첫 번째 층: 선, 모서리 발견       ─┐
                                    │ 점점 복잡한
두 번째 층: 눈, 코, 귀 발견        │ 특징을 인식
                                    │
세 번째 층: "이건 고양이 얼굴!"   ─┘

    가로선, 세로선, 대각선 같은 아주 단순한 패턴을 찾아요. 마치 스케치의 첫 선을 긋는 것처럼요.

    1층에서 찾은 선들을 조합해서 눈, 코, 귀 같은 부분을 인식해요. 선들이 모여 도형이 되는 거죠.

    2층의 부분들을 합쳐서 "이건 고양이 얼굴!", "이건 강아지!" 하고 전체를 판단해요.

    더 알아보기: 합성곱(Convolution)

    이렇게 작은 필터를 이미지 위에서 슬라이딩하면서 특징을 추출하는 연산을 합성곱(Convolution)이라고 해요. 이걸 사용하는 신경망이 바로 CNN(Convolutional Neural Network, 합성곱 신경망)이에요. Teachable Machine도 내부적으로 CNN 기반 모델을 사용하고 있어요!

    합성곱은 수학적으로는 필터 행렬과 이미지 영역의 원소별 곱셈 후 합산인데, 지금은 "작은 창문으로 훑는다"고만 이해하면 충분해요.

    실습: Teachable Machine으로 가위바위보 분류기 만들기¶

    준비물¶

    • 웹캠이 있는 컴퓨터 또는 노트북
    • 인터넷 연결

    Step 1: Teachable Machine 접속¶

    1. 브라우저에서 teachablemachine.withgoogle.com 접속
    2. "Get Started" 클릭
    3. "Image Project" 선택 → "Standard image model" 클릭

    크롬(Chrome) 브라우저를 추천해요

    Teachable Machine은 크롬에서 가장 안정적으로 동작해요. 웹캠 권한을 요청하면 "허용"을 눌러주세요.

    Step 2: 주먹 데이터 수집¶

    1. Class 1의 이름을 "주먹"으로 변경
    2. "Webcam" 버튼 클릭
    3. 웹캠 앞에서 주먹 모양을 만들고 "Hold to Record" 버튼을 꾹 누르기
    4. 다양한 각도로 손을 살짝 돌려가며 최소 20장 촬영

    촬영 팁

    • 배경이 너무 복잡하지 않은 곳에서 촬영해요
    • 손 위치를 살짝씩 바꿔가며 찍어야 다양한 데이터가 모여요
    • 너무 빠르게 움직이면 흔들린 사진이 찍혀요

    Step 3: 손바닥 데이터 수집¶

    1. Class 2의 이름을 "보"로 변경
    2. 같은 방식으로 손바닥(보) 사진 최소 20장 촬영
    3. 역시 다양한 각도로!

    Step 4: 학습시키기¶

    1. "Train Model" 버튼 클릭
    2. 잠시 기다리면 (보통 3~10초) 학습 완료!

    Step 5: 테스트¶

    1. Preview 패널에서 웹캠이 켜져 있는지 확인
    2. 주먹을 보여주면 "주먹"이 높은 확률로 나오는지 확인
    3. 보를 보여주면 "보"가 높은 확률로 나오는지 확인
    잘 안 된다면?
    • 데이터를 더 많이 촬영해 보세요 (각 클래스 40장 이상)
    • 배경을 단순하게 바꿔 보세요
    • 조명이 너무 어둡거나 밝지는 않은지 확인해 보세요
    • 두 클래스의 촬영 환경(배경, 조명)이 비슷한지 확인해 보세요

    Step 6: 도전! 가위 추가하기¶

    1. "Add a class" 클릭해서 Class 3 추가
    2. 이름을 "가위"로 변경
    3. 가위 모양 사진 20장 이상 촬영
    4. 다시 "Train Model" 클릭
    5. 세 가지 모양을 번갈아 보여주며 테스트!

    Step 7: 모둠 활동 — "우리 모둠만의 분류기"¶

    모둠별로 자유 주제를 정해서 분류기를 만들어 봐요!

    연필, 지우개, 자 등을 분류하는 AI

    웃는 얼굴, 놀란 얼굴, 화난 얼굴 분류

    엄지 척, 하트, OK 사인 분류

    교과서, 공책, 핸드폰 분류

    Step 8: 실험 — 편향(Bias) 체험하기¶

    이 실험이 이번 차시에서 가장 중요해요!

    1. 새 프로젝트를 하나 더 만들어요
    2. Class 1 "주먹": 밝은 조명에서만 촬영 (20장)
    3. Class 2 "보": 어두운 환경(손으로 조명을 가리고)에서만 촬영 (20장)
    4. 학습 후 테스트:
      • 밝은 곳에서 "보"를 보여주면? → 엉뚱한 결과!
      • 어두운 곳에서 "주먹"을 보여주면? → 역시 엉뚱한 결과!

    왜 이런 일이 생길까?

    AI가 "주먹 vs 보"를 학습한 게 아니라 "밝음 vs 어두움"을 학습해 버린 거예요! 이게 바로 편향(Bias)이에요. 데이터가 치우치면 AI도 치우친 판단을 해요.

    자주 하는 오해¶

    오해 1: AI가 사진을 '이해'한다?

    AI는 사진의 의미를 이해하는 게 아니에요. 숫자 패턴의 통계적 규칙성을 찾아낼 뿐이에요. "고양이가 뭔지" 모르지만 "이 숫자 패턴이면 고양이라고 답하면 맞을 확률이 높다"는 걸 학습한 거예요.

    오해 2: 데이터가 많으면 무조건 좋다?

    양도 중요하지만 다양성과 품질이 더 중요해요. 똑같은 사진 1,000장보다 다양한 각도의 100장이 훨씬 나아요.

    오해 3: Teachable Machine이 처음부터 학습한다?

    사실 Teachable Machine은 이미 다른 거대한 데이터로 기본 학습이 된 모델 위에 우리 데이터를 추가 학습시키는 거예요. 이걸 전이 학습(Transfer Learning)이라고 해요. 그래서 사진 20장으로도 꽤 잘 되는 거예요!

    정리¶

    **핵심 정리** 1. 컴퓨터에게 사진은 **0~255 숫자로 이루어진 격자(픽셀)**예요 2. **분류(Classification)**란 입력을 받아 카테고리를 정해주는 것이에요 3. AI는 데이터에서 **패턴**을 찾아 분류해요 4. 학습 데이터의 **양, 다양성, 품질**이 결과를 좌우해요 5. **데이터가 바뀌면 결과가 바뀐다** — 이것이 가장 중요한 원리예요
