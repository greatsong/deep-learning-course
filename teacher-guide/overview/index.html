    과목 개요¶

    항목 내용
    과목명 인공지능 기초 — "딥러닝 여행"
    대상 고등학교 2학년

    학생 구성 코딩에 관심 있는 학생 1/3, 어려워하는 학생 1/3, 보통 학생 1/3

    수업 구성 원칙¶

    매 차시는 다음 4단계로 구성됩니다.

    ┌──────────┬──────────┬──────────────┬──────────┐
│  도입     │  이론     │   실습        │  정리     │
│  5분      │  10분    │   25분        │  10분     │
└──────────┴──────────┴──────────────┴──────────┘
    • 도입(5분): 질문, 시연, 놀라운 사실로 호기심 점화
    • 이론(10분): 핵심 개념을 비유와 시각 자료로 설명
    • 실습(25분): 직접 만지고, 관찰하고, 비교하는 활동
    • 정리(10분): 배운 내용 요약, 질문 시간, 다음 차시 예고

    3개 섹션 구조¶

    섹션 1: AI는 글자를 어떻게 읽을까? (0~3차시)¶

    차시 주제 핵심 질문
    0차시 AI, 70년의 꿈 AI는 하루아침에 만들어졌을까?
    1차시 토큰화 AI는 왜 글자 수를 못 셀까?
    2차시 확률과 Temperature AI는 어떻게 다음 단어를 고를까?
    3차시 임베딩 컴퓨터에게 단어는 숫자일까?

    섹션 2: AI는 어떻게 배울까? (4~6차시)¶

    차시 주제 핵심 질문
    4차시 신경망 기초 뉴런이 모이면 지능이 될까?
    5차시 학습과 역전파 AI는 실수에서 어떻게 배울까?
    6차시 과적합과 일반화 시험은 왜 교과서와 다른 문제가 나올까?

    섹션 3: AI는 무엇을 할 수 있을까? (7~9차시)¶

    차시 주제 핵심 질문
    7차시 CNN과 이미지 인식 AI는 사진을 어떻게 이해할까?
    8차시 Transformer와 Attention ChatGPT는 어떤 구조로 만들어졌을까?
    9차시 AI 윤리와 미래 AI 시대, 우리는 무엇을 준비해야 할까?

    평가 방법 (과정 중심 평가)¶

    평가 철학

    결과물의 정확성보다 탐구 과정과 사고의 깊이를 평가합니다.

    평가 요소별 배점¶

    평가 요소 비중 방법 시기
    활동 참여도 30% 관찰 평가, 자기 평가 체크리스트 매 차시
    활동지 40% 차시별 활동지 완성도 1~8차시
    최종 설명서 30% "AI 설명서" 작성 (자유 주제) 9차시

    활동 참여도 평가 기준¶

    등급 기준
    A 적극적으로 질문하고, 모둠원과 협력하며, 자기 생각을 표현함
    B 활동에 참여하고, 모둠 활동에 협력함
    C 활동에 참여하나, 소극적임
    D 활동에 거의 참여하지 않음

    최종 설명서 평가 기준¶

    • 배운 개념 중 1개 이상을 선택하여 "중학생에게 설명하듯" 작성
    • 비유, 그림, 예시 활용 여부
    • 자신의 언어로 재구성한 정도

    수준별 운영 팁¶

    코딩에 관심 있는 학생 (상위 1/3)¶

    심화 전략

    • 실습 시간에 추가 챌린지 제공 (예: "토큰 수를 최소로 줄여보세요")
    • 알고리즘의 원리를 더 깊이 설명하는 보충 자료 안내
    • 모둠 활동에서 설명자 역할 부여 (가르치면서 더 깊이 이해)
    • 관심 있는 학생에게는 수업 후 Python 코드 링크 제공

    어려워하는 학생 (하위 1/3)¶

    지원 전략

    • 코딩 없이 웹 도구만으로 실습 가능하도록 설계됨을 안내
    • 교사가 함께 조작하며 진행 (원격 제어 또는 옆에서 안내)
    • 결과 관찰만으로도 학습 목표 달성 가능함을 강조
    • "틀려도 괜찮아요. 관찰하는 것 자체가 학습이에요" 분위기 조성
    • 모둠 구성 시 상위 학생과 짝 배치

    보통 학생 (중위 1/3)¶

    기본 전략

    • 기본 실습을 완수하는 것을 목표로 설정
    • 활동지의 가이드 질문을 따라가면 자연스럽게 완료되도록 설계
    • 이해가 빠르면 심화 챌린지로 유도

    모둠 구성 원칙¶

    • 4인 1모둠 권장 (상 1명 + 중 2명 + 하 1명)
    • 상위 학생이 "가르치는 역할"을 하되, 일방적 지시가 되지 않도록 안내
    • 모둠 내 역할 분담: 조작자, 기록자, 발표자, 관찰자 (매 차시 순환)
