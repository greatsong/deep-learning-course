# 1차시 교사용 지도서: 토큰화

## 수업 목표

- AI가 텍스트를 글자 단위가 아닌 토큰 단위로 처리한다는 것을 이해한다.
- OpenAI Tokenizer를 사용하여 토큰화를 직접 체험한다.
- 한글과 영어의 토큰화 차이를 관찰하고, 그 원인을 생각한다.

---

## 수업 흐름

| 시간 | 활동 | 세부 내용 |
|------|------|-----------|
| 0~5분 | **도입** | strawberry r 개수, 별표 세기 시연 |
| 5~15분 | **이론** | 토큰화 개념, BPE, 한글 vs 영어 |
| 15~40분 | **실습** | OpenAI Tokenizer + micropgt |
| 40~50분 | **정리** | 핵심 정리, 다음 차시 예고 |

## 도입 시나리오 (0~5분)

### 교사 대사 (그대로 읽어도 됩니다)

> "자, ChatGPT한테 질문 하나 해볼게요."

(화면 공유, ChatGPT 열기)

> "'strawberry에 r이 몇 개야?'... 뭐라고 하죠? 2개래요. 진짜 세어볼까요? s-t-r-a-w-b-e-r-r-y... r이 3개죠?"

(학생들이 반응할 시간을 준다)

> "이번엔 이거."

(별표 30개를 입력: `******************************`)

> "'별표 몇 개야?'... 28개래요. 틀렸어요."

(3초 멈춤, 학생들의 놀라움을 확인)

> "시를 쓰고 코드를 짜는 AI가 글자 수도 못 세는 이유가 뭘까요?"

(3초 멈춤)

> "바보라서? 아닙니다. 이유는... AI는 애초에 글자를 안 보기 때문이에요."

## 학생 예상 반응 & 교사 피드백 (도입 직후)

| 학생 반응 | 교사 피드백 |
|-----------|-------------|
| "AI가 이것도 못해요?" | "못하는 게 아니라 **보는 방식이 다른** 거예요. 오늘 그 방식을 배울 겁니다." |
| "그럼 글자 수를 세는 AI는 없어요?" | "코드로 세면 되지만, GPT는 구조적으로 토큰 단위로 처리해서 글자 수 세기가 어려워요." |
| "토큰이 뭐예요?" | "곧 배울 거예요. 잠깐만 기다려보세요." (호기심을 유지시킨다) |
| "이거 코딩해야 해요?" (불안한 표정) | "아니요! 웹사이트에서 클릭만 하면 돼요. 걱정 마세요." |
| (반응 없음, 조용함) | "놀랍지 않아요? 수능 국어 만점 수준의 AI가 글자 수를 못 세요. 이유가 정말 궁금하지 않아요?" |

## 이론 설명 (5~15분)

### 핵심 개념 전달 순서

#### 1단계: 토큰이란? (3분)

> "AI는 글자를 보지 않아요. '조각'을 봐요. 이 조각을 **토큰**이라고 해요."

**칠판/슬라이드에 예시:**

```
"안녕하세요" → ["안녕", "하", "세요"]     (3토큰)
"Hello"      → ["Hello"]                  (1토큰)
"strawberry" → ["straw", "berry"]          (2토큰)
```

> "보세요. 'strawberry'가 'straw'와 'berry'로 쪼개져요. AI 눈에는 r이 어디 있는지가 아니라, 두 조각으로 보이는 거예요. 그러니 r 개수를 세기 어렵죠."

#### 2단계: BPE 알고리즘 (3분)

> "토큰을 나누는 규칙을 **BPE(Byte Pair Encoding)**라고 해요. 쉽게 말하면, 자주 나오는 글자 조합을 하나로 묶는 거예요."

**비유:**

> "카톡에서 'ㅋㅋㅋ'를 자주 치니까 자동완성이 되잖아요? 비슷해요. 영어에서 'the'가 너무 자주 나오니까 통째로 하나의 토큰이 되는 거예요."

#### 3단계: 한글 vs 영어 (4분)

> "그런데 문제가 있어요. AI는 주로 영어로 훈련됐어요. 그래서 영어는 익숙한 조합이 많고, 한글은 낯선 조합이 많아요."

**칠판/슬라이드에 비교:**

```
"I love you"    → 3 토큰
"사랑해요"       → 5 토큰 (사, 랑, 해, 요 또는 유사한 분절)
```

> "같은 뜻인데 한글이 토큰을 더 많이 써요. AI에게 한국어는 **비싼 언어**인 셈이죠. API를 쓸 때 토큰당 요금이 붙으니까, 진짜로 비싸요."

!!! warning "교사 참고: BPE 깊이 조절"
    BPE 알고리즘의 구체적 절차(빈도 기반 병합 과정)는 **상위 학생에게만** 설명합니다.
    대부분의 학생에게는 "자주 나오는 조합을 묶는다"는 직관만 전달하면 충분합니다.

## 실습 운영 (15~40분)

### 실습 1: OpenAI Tokenizer 탐색 (10분)

**접속:** `platform.openai.com/tokenizer`

#### 단계별 안내

1. **내 이름 넣어보기 (2분)**
    > "먼저 여러분 이름을 넣어보세요. 몇 개의 토큰으로 나뉘나요?"

    !!! tip "교사 팁"
        학생 이름을 넣으면 개인적 연결이 생겨 흥미가 높아집니다.
        "김민준"이 몇 토큰인지, "Minjun Kim"이 몇 토큰인지 비교하게 하세요.

2. **한글 vs 영어 비교 (3분)**
    > "같은 뜻의 문장을 한글과 영어로 넣어보세요. 토큰 수가 어떻게 다른가요?"

    **추천 문장:**
    - "나는 학생입니다" vs "I am a student"
    - "오늘 날씨가 좋다" vs "The weather is nice today"

3. **토큰 수 줄이기 챌린지 (5분)**

    !!! success "모둠 활동: 같은 뜻, 최소 토큰"
        **미션:** "나는 오늘 학교에 갔다"와 **같은 뜻**의 문장을 만들되, **토큰 수를 최소로** 줄여보세요.

        - 모둠별로 도전
        - 가장 적은 토큰 수를 달성한 모둠 발표
        - 교사 포인트: "영어로 바꾸면 토큰이 줄어요. 이게 공정한가요?"

### 실습 2: micropgt 활용 (15분)

> "이제 micropgt에서 토큰화가 실제로 어떻게 작동하는지 더 자세히 봅시다."

**활동 안내:**

- 다양한 문장을 입력하고 토큰화 결과 관찰
- 특수문자, 이모지, 숫자 등도 넣어보기
- 활동지에 관찰 결과 기록

#### 학생이 시도해볼 흥미로운 입력

| 입력 | 관찰 포인트 |
|------|-------------|
| 이모지 (😀🎉) | 이모지도 토큰으로 변환됨 |
| 숫자 (123456789) | 숫자는 어떻게 쪼개지는지 |
| 코드 (`print("hello")`) | 프로그래밍 코드의 토큰화 |
| 반복 문자 (aaaaaaa) | 반복 패턴의 토큰화 |
| 신조어 (갈비탕 300원) | 학습되지 않은 표현의 토큰화 |

## 실습 중 순회 지도 포인트

### 코딩 관심 학생 (상)

> "BPE가 정확히 어떻게 동작하는지 알고 싶어요?"

- 빈도 기반 병합 과정 간단히 설명
- "가장 자주 나오는 글자 쌍을 하나로 합치고, 다시 가장 자주 나오는 쌍을 합치고... 이걸 반복하는 거예요"
- 토큰 효율 분석: "어떤 문장이 왜 토큰이 적은지 분석해보세요"

### 보통 학생 (중)

- 기본 실습(이름 넣기, 한영 비교, 챌린지)을 순서대로 완수하도록 안내
- 활동지 작성 확인

### 어려워하는 학생 (하)

!!! warning "지원 방법"
    - 교사가 옆에서 함께 Tokenizer를 조작
    - "여기에 이름을 넣어볼까요? (함께 타이핑) 보세요, 3개로 쪼개졌죠?"
    - **결과를 관찰하는 것만으로도 충분**함을 안내
    - "토큰 색깔이 다른 거 보이죠? 각 색깔이 하나의 토큰이에요"

## 실습 중 자주 나오는 질문

!!! question "왜 같은 단어인데 위치에 따라 토큰이 달라져요?"
    "좋은 관찰이에요! 단어 앞에 공백이 있으면 토큰이 달라질 수 있어요. 'the'와 ' the'(앞에 공백)가 다른 토큰이에요. 컴퓨터에게는 공백도 중요한 정보거든요."

!!! question "토큰 수가 많으면 안 좋은 건가요?"
    "토큰이 많으면 AI가 처리할 양이 늘어나요. 그만큼 느려지고, API 비용도 올라가죠. 그래서 효율적인 토큰화가 중요해요."

!!! question "한글 AI를 만들면 한글 토큰이 줄어들지 않나요?"
    "정확해요! 한글을 많이 학습한 모델은 한글 토큰이 더 효율적이에요. 네이버의 HyperCLOVA 같은 한국어 모델이 그런 접근이에요."

!!! question "GPT-4는 이 문제를 해결했나요?"
    "GPT-4도 토큰 기반이라 근본적으로는 같아요. 다만 토큰 사전이 더 커지고 다국어 지원이 좋아져서 한글 효율이 조금 나아졌어요."

## 정리 (40~50분)

### 핵심 정리 (5분)

칠판/슬라이드에 3가지 핵심을 정리합니다.

> **오늘 배운 것:**
>
> 1. AI는 글자가 아니라 **토큰**을 본다
> 2. 토큰은 **자주 나오는 조합**을 기준으로 나뉜다 (BPE)
> 3. 한글은 영어보다 토큰이 많이 필요하다 → **AI에게 한국어는 비싼 언어**

### 연결 질문 (3분)

> "자, 이제 AI가 토큰을 쪼개는 건 알겠어요. 그런데 다음 단계가 궁금하지 않아요? 토큰을 쪼갠 다음에 뭘 할까요? 다음 단어를 어떻게 고를까요?"

(학생 반응을 듣는다)

> "AI가 다음 단어를 고르는 방법은... **확률**이에요. 주사위를 굴려요. 근데 이 주사위를 조절하는 슬라이더가 있어요. Temperature라고 하는데... 이건 다음 시간에!"

### 다음 차시 예고 (2분)

> "다음 시간 제목은 '확률과 Temperature'예요. ChatGPT에 같은 질문을 해도 매번 답이 다른 이유, 궁금하지 않아요?"
