    도입: 한 글자씩 나오는 이유¶

    ChatGPT에게 질문하면, 답변이 한 글자씩 또르르 나오는 거 봤죠?

    질문: "피타고라스 정리를 설명해줘"

답변이 나오는 과정:
  피 → 피타 → 피타고 → 피타고라 → 피타고라스 →
  피타고라스 정 → 피타고라스 정리 → 피타고라스 정리는 → ...

    왜 한꺼번에 "짠!" 하고 전체 답을 보여주지 않을까요?

    그건 AI가 실제로 한 토큰씩 답을 만들고 있기 때문이에요.

    1. "다음에 올 토큰 하나"를 예측해요
    2. 그 토큰을 출력하고, 지금까지의 텍스트에 추가해요
    3. 추가된 텍스트를 보고, 또 "다음 토큰 하나"를 예측해요
    4. 이걸 끝날 때까지 반복해요

    마치 자동완성을 수백 번 연속으로 누르는 것과 같아요!

    오늘은 이 과정의 전체 그림을 조립할 거예요. 그리고 놀라운 사실 — 여러분은 이미 각 조각을 전부 배웠어요.

    개념 1: GPT의 T = Transformer¶

    ChatGPT의 이름을 풀어볼게요:

    글자 의미 풀이
    G Generative 생성하는 — 텍스트를 만들어내는 AI
    P Pre-trained 미리 학습된 — 엄청난 양의 텍스트로 미리 훈련받은
    T Transformer 트랜스포머 — 우리가 지금 배우고 있는 모델 구조

    이제 T가 뭔지 아는 사람이 됐어요

    대부분의 사람들은 ChatGPT를 쓰면서도 T가 뭔지 몰라요. 하지만 여러분은 이제 Transformer가 어떻게 작동하는지 속까지 아는 사람이 됐어요. 이건 꽤 대단한 거예요!

    개념 2: Transformer의 전체 흐름 — 1~8차시 조립!¶

    자, 이제 진짜 조립을 해 볼게요. 여러분이 ChatGPT에 "오늘 날씨 어때?"라고 입력하면 내부에서 무슨 일이 벌어지는지, 한 단계씩 따라가 봐요.

    **Transformer 전체 흐름도** 입력 → 토큰화(1차시) → 임베딩(3차시) → 위치 인코딩(NEW) → Attention(8차시) → 신경망(5차시) → 확률 계산(2차시) → Temperature(2차시) → 토큰 출력 → 다시 처음으로!

    Step 1: 텍스트 → 토큰화 (1차시)¶

    입력된 문장을 숫자 조각(토큰)으로 분해해요.

    "오늘 날씨 어때?"
→ ["오늘", " 날", "씨", " 어때", "?"]
→ [8834, 1127, 3892, 5541, 30]

    1차시에서 배웠죠? AI는 글자를 직접 이해 못 하니까, 먼저 숫자로 바꿔야 해요.

    Step 2: 토큰 → 임베딩 (3차시)¶

    각 토큰이 고차원 공간의 좌표(벡터)를 받아요.

    "오늘" → [0.23, -0.15, 0.87, 0.42, ...]  (수백~수천 차원)
"날"   → [0.45, 0.12, -0.33, 0.67, ...]
"씨"   → [0.41, 0.18, -0.29, 0.59, ...]

    3차시에서 배운 임베딩이에요! "왕 - 남자 + 여자 = 여왕"이 가능했던 그 벡터 공간이요.

    Step 3: 위치 인코딩 — 순서 정보 추가¶

    새로운 개념! (하지만 간단해요)

    Transformer는 모든 단어를 동시에 처리하기 때문에, 그냥 두면 단어의 순서를 몰라요. 그래서 각 위치에 고유한 숫자 패턴을 더해줘요.

    왜 순서가 중요할까요?

    "나는 너를 좋아해" vs "너를 나는 좋아해"

    의미가 미묘하게 다르죠? 순서가 바뀌면 뉘앙스가 달라져요.

    "오늘"의 임베딩:  [0.23, -0.15, 0.87, ...]
+ 1번째 위치 패턴: [0.01,  0.00, 0.01, ...]
= 위치가 반영된 벡터: [0.24, -0.15, 0.88, ...]

    이렇게 각 단어에 "나는 몇 번째야!"라는 정보를 살짝 더해주는 거예요.

    Step 4: Attention × 여러 번 (8차시)¶

    8차시에서 배운 Self-Attention이 작동해요! 각 단어가 다른 모든 단어를 살펴보면서, "나를 이해하려면 어디를 봐야 하지?"를 계산해요.

    "어때"가 주목하는 곳:
  오늘  ████████████████  40%
  날    ██████████████    35%
  씨    ████████          20%
  어때  ██                 5%

    그런데 Transformer는 이걸 한 번만 하지 않아요.

    사람도 같은 문장을 여러 관점에서 볼 수 있잖아요?

    • 관점 1 (문법 안경): "어때"는 의문형이니까 → "날씨"에 집중
    • 관점 2 (주제 안경): 이 문장의 주제는? → "오늘"에 집중
    • 관점 3 (감정 안경): 감정은? → "어때"의 물음표에 집중

    마치 여러 개의 안경을 동시에 쓰고 문장을 보는 것 같아요. 이걸 Multi-Head Attention이라고 해요. 각 "Head"가 다른 관점에서 Attention을 계산하고, 결과를 합쳐요.

    Step 5: Feed Forward 신경망 (5차시)¶

    Attention으로 관계를 파악한 뒤, 그 정보를 신경망에 통과시켜요. 5차시에서 배운 뉴런들이 여기서 활약해요! 입력을 받아서 → 가중치를 곱하고 → 활성화 함수를 통과하는 그 과정이요.

    이 신경망이 Attention에서 모은 정보를 더 깊이 처리해줘요.

    Step 4와 5를 여러 번 반복해요

    Transformer는 Attention → 신경망을 여러 층(layer) 쌓아요. GPT-3는 96층, GPT-4는 더 많을 거예요. 층을 많이 쌓을수록 더 복잡한 패턴을 파악할 수 있어요.

    Step 6: 확률 계산 (2차시)¶

    모든 처리가 끝나면, 다음에 올 토큰의 확률 분포를 만들어요. 2차시에서 배운 바로 그 확률이에요!

    "오늘 날씨 어때?" 다음에 올 토큰:

오늘    ████████████████████  25%
날씨가  ████████████████      20%
좋      █████████████         15%
맑      ██████████            12%
흐      ████████               9%
비      ██████                 7%
...     ████                   (나머지)

    Step 7: Temperature로 토큰 선택 (2차시)¶

    2차시에서 배운 Temperature가 여기서 작동해요!

    • Temperature 낮음 → 확률 1등 "오늘"만 선택 (정확하지만 뻔함)
    • Temperature 적당 → "좋" 같은 자연스러운 답도 선택 가능
    • Temperature 높음 → "비" 같은 의외의 답도 나올 수 있음

    Step 8: 출력하고, 다시 처음으로!¶

    선택된 토큰을 출력하고, 기존 입력에 추가해요. 그리고 Step 1로 돌아가요.

    1회차 입력: "오늘 날씨 어때?"
1회차 출력: "오늘"

2회차 입력: "오늘 날씨 어때? 오늘"
2회차 출력: "날씨는"

3회차 입력: "오늘 날씨 어때? 오늘 날씨는"
3회차 출력: "맑고"

... (종료 토큰이 나올 때까지 반복)

    이게 바로 ChatGPT의 답이 한 글자씩 나오는 이유예요! 진짜로 한 토큰씩 만들고 있거든요.

    그리고 학습 과정에서는...¶

    이 전체 과정이 제대로 작동하려면, 모델을 학습시켜야 해요. 학습할 때는:

    • 손실함수 (6차시): 모델의 예측이 정답과 얼마나 다른지 측정해요
    • 경사하강법 (7차시): 손실을 줄이는 방향으로 파라미터를 조금씩 조정해요

    이 과정을 수억 번 반복하면, Transformer가 점점 더 자연스러운 텍스트를 생성하게 돼요.

    개념 3: 왜 Transformer가 혁명인가¶

    • 단어를 순서대로 하나씩 처리
    • 앞 단어 처리가 끝나야 다음 단어 처리 가능 → 느림
    • 문장이 길면 앞부분을 까먹음
    • GPU가 있어도 순서대로 해야 하니 병렬 처리 불가
    • 모든 단어를 동시에 처리
    • Attention으로 어떤 단어든 직접 참조 가능 → 빠름
    • 긴 문장도 Attention으로 직접 연결 → 안 까먹음
    • 동시 처리니까 GPU 병렬 처리 최적화 → 학습 속도 폭발

    비유: 줄 서기 vs 동시 입장

    RNN은 놀이공원 줄 서기예요. 앞 사람이 타야 다음 사람이 탈 수 있어요. Transformer는 대형 콘서트장이에요. 모든 관객이 동시에 입장해서 자기 자리에 앉아요. 당연히 Transformer가 훨씬 빠르겠죠?

    2017년 Transformer가 등장한 이후, 거의 모든 주요 AI 모델이 Transformer 기반이에요:

    모델 만든 곳 기반
    GPT-4 OpenAI Transformer
    Gemini Google Transformer
    Claude Anthropic Transformer
    Llama Meta Transformer

    개념 4: 규모의 힘 (Scaling)¶

    Transformer의 구조 자체는 2017년 이후 크게 바뀌지 않았어요. 그런데 성능은 어떻게 계속 좋아질까요?

    답: 크기를 키웠어요.

    GPT-1 (2018):   1억 파라미터      █
GPT-2 (2019):  15억 파라미터      ██
GPT-3 (2020): 1750억 파라미터     ████████████████████
GPT-4 (2023): 수조 파라미터 (추정) ████████████████████████████████████████

    같은 Transformer 구조인데, 파라미터(7차시에서 배운 가중치의 개수)를 엄청나게 늘렸더니 놀라운 일이 벌어졌어요.

    창발적 능력 (Emergent Abilities):

    • GPT-2: 짧은 글을 이어 쓸 수 있음
    • GPT-3: 갑자기 번역, 요약, 질의응답이 가능해짐
    • GPT-4: 시를 쓰고, 코드를 짜고, 수학 문제를 풀기 시작

    규모를 키우니까 가르치지 않은 능력이 갑자기 나타나기 시작한 거예요. 마치 물을 계속 데우면 어느 순간 갑자기 끓기 시작하는 것처럼요.

    흔한 오해: AI가 이해한다?

    AI의 능력이 아무리 뛰어나 보여도, 그 안에서 일어나는 건 우리가 배운 것과 같아요 — 토큰화 → 임베딩 → Attention → 신경망 → 확률 예측. AI가 "이해"하는 것처럼 보이지만, 실제로는 엄청나게 정교한 패턴 매칭일 수 있어요. "진짜 이해"와 "패턴 매칭"의 경계는 아직 과학자들도 논쟁 중인 주제예요.

    실습 안내¶

    실습 1: Transformer 구조도 완성하기¶

    빈 Transformer 구조도 활동지를 채워 봐요!

    1. 선생님이 빈 블록이 있는 Transformer 구조도를 나눠줘요
    2. 각 블록에 해당 차시 번호와 개념 이름을 채워넣어요
    3. 데이터가 흘러가는 방향에 화살표를 그려요
    4. 마지막에 "출력 → 다시 입력"으로 돌아가는 순환 화살표도 잊지 마요!

    채워넣을 블록들:

    블록 차시 개념
    텍스트 → 숫자 조각 1차시 토큰화
    숫자 → 벡터 공간 3차시 임베딩
    순서 정보 추가 9차시 위치 인코딩
    단어 간 관계 파악 8차시 Attention
    정보 깊이 처리 5차시 신경망
    다음 토큰 확률 2차시 확률 분포
    토큰 선택 방식 2차시 Temperature
    오차 측정 6차시 손실함수
    가중치 업데이트 7차시 경사하강법

    실습 2: "내가 만든 Transformer 설명서"¶

    A4 한 장에 1~8차시 핵심 개념을 연결해서 Transformer를 설명해 봐요.

    규칙:

    • 그림, 화살표, 비유 자유롭게 사용
    • 전문 용어를 쓸 때는 반드시 쉬운 설명도 함께
    • "초등학생 동생에게 설명한다"는 마음으로!
    • 모둠별 발표 (2분)

    잘 쓰는 팁

    "AI에게 글을 주면, 먼저 로 쪼개고, 그 다음 로 바꾸고, ___로 어디를 볼지 정하고..."처럼 흐름으로 쓰면 이해하기 쉬워요.

    실습 3: ChatGPT에게 "넌 어떻게 작동해?" 물어보기¶

    재미있는 실험이에요!

    1. ChatGPT (또는 다른 AI)에게 물어봐요: "넌 어떻게 작동해? 쉽게 설명해줘."
    2. AI의 자기 설명을 읽어봐요
    3. 우리가 배운 내용과 비교해요:
      • AI가 정확하게 설명한 부분은?
      • AI가 빠뜨리거나 부정확하게 설명한 부분은?
      • AI가 자기 자신에 대해 모르는 것도 있나요?

    생각해 볼 점

    AI가 자기 자신의 작동 원리를 설명한다고 해서, 그게 정확하다는 보장은 없어요. AI는 학습 데이터에서 "AI 작동 원리"에 대한 글을 읽고 그걸 패턴으로 재현하는 거니까요. 여러분이 직접 배운 지식으로 AI의 설명을 검증할 수 있다는 게 중요해요!

    자주 하는 오해¶

    오해 1: Transformer는 한 번에 전체 답을 만든다?

    아니에요! Transformer는 한 번에 토큰 하나만 예측해요. 긴 답변도 토큰을 하나씩 이어 붙여서 만드는 거예요. 그래서 ChatGPT의 답이 한 글자씩 나오는 거예요.

    오해 2: GPT-4가 GPT-3보다 좋은 건 구조가 다르기 때문이다?

    꼭 그렇지는 않아요! 기본 구조(Transformer)는 크게 다르지 않아요. 주된 차이는 규모(파라미터 수, 학습 데이터 양)와 학습 방법의 개선이에요. 같은 레시피로 케이크를 굽되, 재료를 1000배 더 넣은 것에 가까워요.

    오해 3: AI가 글을 쓸 때 전체 계획을 세우고 쓴다?

    아니에요! AI는 항상 "바로 다음 토큰"만 예측해요. 전체 글의 구조를 미리 계획하지 않아요. 그런데 신기하게도 Attention 덕분에 앞에서 쓴 내용을 참고할 수 있어서, 결과적으로 일관성 있는 글이 나오는 거예요.

    정리¶

    전체 흐름 한눈에 보기¶

    ┌─────────────────────────────────────────────────┐
│                  Transformer 전체 흐름             │
├─────────────────────────────────────────────────┤
│                                                   │
│  "오늘 날씨 어때?"                                │
│       ↓                                          │
│  ① 토큰화 (1차시)     → 숫자 조각으로 분해         │
│       ↓                                          │
│  ② 임베딩 (3차시)     → 벡터 공간에 배치           │
│       ↓                                          │
│  ③ 위치 인코딩        → 순서 정보 추가             │
│       ↓                                          │
│  ┌─────────────────┐  ← 이 블록을 여러 번 반복    │
│  │ ④ Attention (8차시) → 단어 간 관계 파악       │ │
│  │ ⑤ 신경망 (5차시)    → 정보 깊이 처리          │ │
│  └─────────────────┘                             │
│       ↓                                          │
│  ⑥ 확률 계산 (2차시)  → 다음 토큰 확률 분포       │
│       ↓                                          │
│  ⑦ Temperature (2차시) → 토큰 선택               │
│       ↓                                          │
│  ⑧ 토큰 출력 → 입력에 추가 → ①로 돌아감!         │
│                                                   │
│  [학습 시: 손실함수(6차시) + 경사하강법(7차시)]     │
│                                                   │
└─────────────────────────────────────────────────┘

    핵심 용어 총정리¶

    차시 개념 Transformer에서의 역할
    1차시 토큰화 텍스트를 숫자 조각으로 분해
    2차시 확률/Temperature 다음 토큰 선택 방식 결정
    3차시 임베딩 토큰을 벡터 공간에 배치
    5차시 신경망 Attention 결과를 깊이 처리
    6차시 손실함수 예측과 정답의 차이 측정
    7차시 경사하강법 파라미터를 조금씩 개선
    8차시 Attention 단어 간 관계 파악 (핵심!)
    9차시 위치 인코딩 단어 순서 정보 추가
    **오늘의 핵심 한 줄** Transformer = 토큰화 + 임베딩 + Attention + 신경망 + 확률 예측. 우리가 배운 모든 것의 조립이에요.
    **마무리 메시지** 여러분은 이제 ChatGPT의 **겉이 아닌 속**을 아는 사람이에요. 대부분의 어른들도 모르는 Transformer의 작동 원리를, 여러분은 토큰화부터 Attention까지 하나씩 직접 이해했어요. 이건 AI를 **도구로 쓰는 사람**과 **원리를 아는 사람**의 차이예요. 앞으로 어떤 새로운 AI가 나와도, 이 기초 위에서 이해할 수 있을 거예요.
