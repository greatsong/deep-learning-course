# 4차시: Teachable Machine으로 나만의 AI 만들기

## 1. 수업 개요

### 학습 목표

- 학생이 직접 이미지 분류 AI를 만들어보고, **데이터가 결과를 바꾼다**는 것을 체험한다.
- 학습 데이터의 양과 질이 AI 성능에 미치는 영향을 실험으로 확인한다.
- 편향된 데이터가 만드는 편향된 AI를 직접 경험한다.

### 준비물

| 항목 | 수량 | 비고 |
|------|------|------|
| 노트북 + 웹캠 | 모둠당 1대 | Chrome 브라우저 필수 |
| WiFi | - | 30명 동시 접속 확인 |
| 프로젝터/대형 모니터 | 1대 | 교사 시연용 |
| 활동지 | 1인 1장 | "우리 모둠만의 분류기" 기록용 |

!!! warning "사전 확인"
    `teachablemachine.withgoogle.com` 접속이 학교 네트워크에서 차단되지 않는지 전날 확인하세요. 카메라 권한 허용도 미리 테스트하세요.

### 시간 배분

| 단계 | 시간 | 활동 내용 |
|------|------|-----------|
| 도입 | 5분 | 교사 시연 - 10초 AI 만들기 |
| 이론 전개 | 8분 | 데이터 -> 학습 -> 분류 흐름 설명 |
| 실습 1 | 15분 | 모둠별 "우리 모둠만의 분류기" 제작 |
| 실습 2 | 12분 | 편향 실험 + 발표 |
| 정리 | 10분 | 핵심 정리 + 다음 차시 예고 |

## 2. 도입 시나리오 (5분)

### 교사 대사 전문

> "여러분, 선생님이 지금 AI를 만들 건데요, 10초면 됩니다. 진짜인지 보세요."

**[화면: teachablemachine.withgoogle.com 접속, Image Project > Standard image model 선택]**

> "자, 여기 Class 1에 '주먹'이라고 이름을 붙이고... 웹캠으로 주먹 사진을 찍을게요."

**[주먹을 보여주며 연속 촬영 — 약 10장]**

> "이번엔 Class 2에 '손바닥'... 손바닥도 10장."

**[손바닥을 보여주며 연속 촬영 — 약 10장]**

> "이제 Train Model 버튼을 누르면... 끝! 자, 보세요."

**[Preview 패널에서 주먹/손바닥 번갈아 보여주며 실시간 인식 시연]**

> "주먹 — 98%, 손바닥 — 97%. 이게 AI예요. 여러분이 찍은 사진이 AI의 '교과서'가 된 거예요. 오늘은 여러분이 직접 이걸 만들어볼 겁니다."

!!! tip "시연 팁"
    시연은 **최대한 빠르게, 거침없이** 진행하세요. 학생들의 "와, 진짜?" 반응을 이끌어내는 것이 핵심입니다. 미리 2~3번 연습해 두면 10초 안에 가능합니다.

## 3. 이론 전개 포인트 (8분)

### 설명 순서

**1단계: 데이터 수집 = AI의 교과서 만들기**

> "여러분이 시험공부할 때 교과서를 보죠? AI도 마찬가지예요. 우리가 찍은 사진이 AI의 교과서입니다. 교과서가 얇으면? 시험을 잘 못 보겠죠."

**2단계: 학습(Training) = 패턴 찾기**

> "Train 버튼을 누르면 AI가 '주먹은 이렇게 생겼고, 손바닥은 이렇게 생겼구나'를 기억하는 게 아니에요. **패턴**을 찾는 거예요. 주먹은 동그랗고 손가락이 안 보이는 패턴, 손바닥은 넓적하고 손가락이 펼쳐진 패턴."

**3단계: 분류(Prediction) = 어디에 가까운지 판단**

> "새로운 사진이 들어오면 AI는 '이건 주먹 패턴에 더 가깝네' 또는 '손바닥 패턴에 더 가깝네'라고 판단합니다. 그래서 확률이 나오는 거예요."

### 강조점

!!! info "핵심 메시지"
    **"데이터가 달라지면 AI도 달라진다."** 이 문장을 칠판에 크게 쓰거나 슬라이드에 띄워두세요. 오늘 수업의 모든 활동이 이 문장으로 귀결됩니다.

## 4. 실습 운영 상세

### 실습 1: 우리 모둠만의 분류기 (15분)

#### 추천 주제 (모둠별 자유 선택)

| 주제 | 클래스 예시 | 난이도 |
|------|------------|--------|
| 가위바위보 | 가위 / 바위 / 보 | 쉬움 |
| 표정 분류 | 웃음 / 무표정 / 찡그림 | 보통 |
| 필통 속 물건 | 펜 / 지우개 / 자 | 보통 |
| 손가락 숫자 | 1 / 2 / 3 / 4 / 5 | 어려움 |

#### 단계별 진행

| 단계 | 시간 | 활동 | 교사 역할 |
|------|------|------|-----------|
| 1. 주제 선정 | 2분 | 모둠 회의로 주제 결정 | 빠르게 결정하도록 독려 |
| 2. 데이터 촬영 | 5분 | 클래스당 최소 30장 촬영 | 순회하며 촬영 상태 확인 |
| 3. 학습 | 2분 | Train Model 클릭 후 대기 | "학습 중 화면을 관찰하세요" 안내 |
| 4. 테스트 | 3분 | 다양한 상황에서 테스트 | "다른 배경, 다른 거리에서 해보세요" |
| 5. 발표 준비 | 3분 | 결과 정리, 성공/실패 사례 기록 | 활동지 작성 확인 |

!!! tip "촬영 팁 안내"
    학생들에게 미리 안내하세요: "한 각도에서만 찍지 말고, **살짝 돌리고, 가까이, 멀리**에서도 찍으세요. AI가 더 다양한 패턴을 배울 수 있어요."

### 실습 2: 편향 실험 (12분)

> "자, 이제 일부러 AI를 망가뜨려 봅시다!"

#### 실험 설계

1. **편향 데이터 수집** (3분): 창가 밝은 곳에서**만** 모든 사진을 촬영
2. **학습** (2분): Train Model
3. **테스트** (3분): 복도, 교실 안쪽 등 **어두운 곳**에서 테스트 → 높은 확률로 실패
4. **토론 + 발표** (4분): "왜 실패했을까?" → "데이터가 편향되면 AI도 편향된다"

#### 기대 결과

> "밝은 곳에서만 찍었으니까, AI는 '밝기'를 기준으로 분류하게 됐어요. 손 모양이 아니라요. 이게 바로 **데이터 편향**이에요."

## 5. 학생 예상 반응 & 교사 피드백

### 도입 단계

| 학생 반응 | 교사 피드백 |
|-----------|------------|
| "이게 진짜 AI예요?" | "네! 단순하지만 진짜 AI예요. 사진에서 패턴을 찾아 분류하는 거니까요. 여러분도 지금 바로 만들 수 있어요." |
| "ChatGPT랑 다른 거예요?" | "ChatGPT는 텍스트를 다루는 AI고, 이건 이미지를 분류하는 AI예요. 원리는 비슷하지만 하는 일이 달라요." |
| "와 신기하다!" (코딩 관심 학생) | "오늘 직접 만들면 더 신기할 거예요. 심화로 모델 내보내기까지 해볼 수 있어요." |

### 실습 단계

| 학생 반응 | 교사 피드백 |
|-----------|------------|
| "웹캠이 안 돼요" | "크롬 주소창 왼쪽 자물쇠 아이콘 클릭 → 카메라 → 허용으로 바꿔주세요. 그래도 안 되면 크롬을 완전히 닫았다 다시 열어보세요." |
| "학습이 안 돼요 / 결과가 이상해요" | "사진 수를 확인해 보세요. 클래스당 최소 20장은 있어야 해요. 배경이 너무 비슷하면 AI가 손이 아니라 배경으로 분류할 수 있어요." |
| "가위를 추가했는데 잘 안 돼요" | "각 클래스당 데이터를 **비슷한 양**으로 맞춰주세요. 주먹 50장, 가위 5장이면 AI가 뭐든 주먹이라고 할 거예요." |
| "왜 뒤집으면 안 되죠?" | "AI는 패턴으로 분류하니까 각도가 크게 바뀌면 다른 패턴으로 인식해요. 그래서 다양한 각도로 촬영하는 게 중요한 거예요." |
| "이걸 앱으로 만들 수 있어요?" (코딩 관심 학생) | "네, Export Model 버튼으로 모델을 내보내서 웹이나 앱에 넣을 수 있어요. 관심 있으면 수업 후에 링크 알려줄게요." |
| "너무 쉬운데요, 더 할 거 없어요?" | "5개 클래스까지 늘려보세요. 그리고 Preview의 확률값이 어떻게 변하는지 관찰해 보세요." |
| "우리 모둠은 뭘 해야 해요?" (결정 못하는 모둠) | "가위바위보가 가장 쉬워요. 먼저 그걸로 시작하고, 시간 남으면 다른 주제로 바꿔도 돼요." |

## 6. 수준별 분화 전략

### 상위 수준 (코딩 관심 학생)

!!! tip "심화 활동"
    - **모델 내보내기**: Export Model → TensorFlow.js로 내보내기 체험
    - **확률값 분석**: Preview 패널의 확률 바 관찰 — "왜 98%가 아니라 73%일까?"
    - **추가 클래스 실험**: 5개 이상 클래스로 확장 시 정확도 변화 관찰
    - **데이터 수 실험**: 10장 vs 50장 vs 100장의 성능 차이 비교

### 중위 수준 (보통 학생)

!!! note "기본 목표"
    - 기본 **3클래스 분류기** 완성 (가위바위보 등)
    - 클래스당 30장 이상 촬영
    - 편향 실험 참여 및 결과 관찰
    - 활동지의 가이드 질문에 답하며 정리

### 하위 수준 (어려워하는 학생)

!!! warning "지원 방안"
    - 교사 또는 상위 학생과 함께 **2클래스부터** 시작 (주먹/손바닥)
    - **성공 체험 우선**: 2클래스가 잘 되면 칭찬 후 3클래스로 확장
    - 직접 조작이 어려우면 **촬영 역할**만 맡겨도 OK ("네가 포즈를 취해줘")
    - "결과를 관찰하고 활동지에 적는 것만으로도 훌륭해요"

## 7. 자주 나오는 질문 FAQ

??? question "Q1. Teachable Machine은 무료인가요?"
    네, Google에서 제공하는 완전 무료 도구입니다. 회원가입도 필요 없습니다. 다만 학습된 모델은 브라우저를 닫으면 사라지므로, 저장하려면 Export를 해야 합니다.

??? question "Q2. 웹캠이 없는 노트북은 어떻게 하나요?"
    스마트폰으로 사진을 찍어 업로드할 수 있습니다. Teachable Machine의 각 클래스에서 Upload 옵션을 선택하면 됩니다. 또는 외장 웹캠을 연결할 수 있습니다.

??? question "Q3. 학습 시간이 너무 오래 걸려요."
    이미지 수가 너무 많거나(클래스당 200장 이상) 노트북 사양이 낮으면 오래 걸릴 수 있습니다. 클래스당 30~50장이면 충분합니다. Chrome 외 다른 탭을 모두 닫아보세요.

??? question "Q4. 이 수업에서 코딩을 해야 하나요?"
    아닙니다. Teachable Machine은 코딩 없이 클릭과 촬영만으로 AI를 만드는 도구입니다. 코딩에 관심 있는 학생에게는 모델 내보내기 후 JavaScript 코드 연동을 안내할 수 있습니다.

??? question "Q5. 학생들이 부적절한 이미지를 촬영하면 어떻게 하나요?"
    수업 시작 전에 "수업 목적에 맞는 이미지만 촬영합니다"라는 규칙을 명확히 안내하세요. 순회하며 모니터링하되, 웹캠 실시간 촬영이므로 저장되지 않는다는 점을 설명해 주세요.

## 8. 정리 & 다음 차시 연결 멘트 (10분)

### 오늘 배운 핵심 정리

> "오늘 우리가 한 일을 정리해 볼게요."

| 단계 | 우리가 한 것 | AI 용어 |
|------|-------------|---------|
| 사진 촬영 | 주먹/손바닥 사진 찍기 | **데이터 수집** |
| Train 클릭 | AI가 패턴을 찾게 하기 | **학습(Training)** |
| 테스트 | 새 사진으로 확인하기 | **추론(Inference)** |
| 편향 실험 | 한쪽 데이터만 넣기 | **데이터 편향(Bias)** |

> "가장 중요한 건 뭐였죠? **데이터가 달라지면 AI도 달라진다.** 이거 하나만 기억하면 오늘 수업 성공이에요."

### 다음 차시 연결

> "그런데 여러분, Teachable Machine 안에서 실제로 무슨 일이 벌어지고 있었을까요? Train 버튼을 누르면 컴퓨터가 뭘 하는 걸까요? 다음 시간에는 이 **AI의 속을 뜯어볼** 거예요. '퍼셉트론'이라는 아주 작은 뇌세포 하나에서 시작합니다. 기대하세요!"

### 마무리 활동

- 활동지 수거 또는 사진 촬영 후 LMS 업로드
- 자기 평가 체크리스트 작성 (1분)
    - [ ] 나는 AI 분류기를 직접 만들어 보았다
    - [ ] 데이터의 양이 AI 성능에 영향을 준다는 것을 알게 되었다
    - [ ] 편향된 데이터의 위험성을 이해했다
